**Sonya Huang**

今天我们的嘉宾是 Prime Intellect 的联合创始人 Will Brown 和 Johannes Hagemann。Prime Intellect 是 AI 领域最酷的新实验室之一，他们的使命是让前沿训练基础设施对所有人开放，这非常了不起。你们推出的 RL environment hub 也极为独特，大家都非常兴奋。我们来聊聊 post-training、reinforcement learning、agent harnesses，以及你们对未来的展望。

**Johannes Hagemann**

我们的平台覆盖从计算层到完整 post-training stack 的一切——包括大规模 RL 训练框架、以社区方式运作的 environment hub、安全代码执行沙箱和 evaluations。

**Sonya Huang**

你们为什么追求把所有这些基础设施开放给所有人的使命？

**Will Brown**

端到端优化系统的方法就是直接访问模型权重，把模型塑造得最适合你的问题，而不是用 off-the-shelf 模型、只在 prompt 层面做定制。这比 prompt 层面的定制要深得多。

## 产品-模型优化循环

**Sonya Huang**

你们的未来愿景是每家公司都会 pre-train 和 post-train 自己的模型吗？

**Will Brown**

我们认为每家公司都会成为 AI 公司，大多数 AI 公司会想要自己的 AI research lab。研究可以是 pre-training，特别是当你的领域不只是 text-in text-out 时；在很多情况下，它会是 post-training 那种 agentic 的、聚焦特定任务和 workflow 的模型。

**Johannes Hagemann**

现在已经到了可以量产、成本可控的阶段，可以大规模地解决人们实际想要解决的问题了。

**Will Brown**

我们的平台叫 Lab，核心概念是 environment。RL environment 封装了做 reinforcement learning 所需的一切——模型通过试错来改进。但它不止于 RL。

本质上，environment 跟新模型发布时报告的那些 eval 是一回事——SWE-bench、AMIE、Terminal Bench 都是 environment 的例子。它们有任务数据集、有 harness、有 rubric 或 reward function 给输出质量打分。你在 offline eval 里做 test set 用的东西，也可以在 RL 里当 training set 用。

我们的平台强调社区概念，像 GitHub 那样——让人们有大量积木可以借鉴，展示他们如何使用不同工具和 workflow，让其他人也能 post-train 模型。

**Johannes Hagemann**

核心思路是给更多公司大实验室才有的产品-模型优化循环。ChatGPT 是 OpenAI 创造的，Claude Code 是 Anthropic 创造的——他们有能力针对自己的 scaffold 优化模型。这些产品越流行，大实验室就越没动力让模型在其他产品上也表现得好。

Cursor 是个很好的例子——他们很早就意识到了，自己做了大规模 post-training，而那个 environment 就是 Cursor 本身。我们相信会有更多 startup 朝这个方向走，构建出之前不可能的产品。

## Environment 与 Eval 的关系

**Sonya Huang**

你说 environments 就是 evals，但我认为 environment 是有状态的——你观察它，采取行动，观察状态变化，更新 world model。这跟 eval 显然不同，eval 像是你应该在这套题目上得到这个答案。你能帮我把这两种现实合并起来吗？

**Will Brown**

一两年以前，很多 eval 是问答形式，就是一大堆问题。同时人们对 environment 还有别的理解，比如老式 RL 和 Atari——长期运行的状态交互循环。但现在两者已经合为一体了，特别是大规模训练用的 environment，它们有复杂的状态，可能模拟一个 web 应用或完整的 coding platform。

跟原始 RL 游戏一样，它们永远有一个 reward 和一个目标。你不需要让人拍脑袋说好不好，而是有东西可以衡量进度和表现——这就是我说它也是 eval 的意思。系统从某个状态开始，跟环境交互，总有一个目标，有办法衡量它做得好不好。

## Agent Harness 与 Environment 的边界

**Sonya Huang**

Agent harness 和 RL，这两个概念是正交的还是相互排斥的？它们怎么关联？

**Will Brown**

Harness 是 environment 的一部分。对任何 eval 或 environment 任务来说，有输入，中间一堆事情发生，然后输出某个状态被评分。environment 就是这个中间部分——跟 simulator 交互、跟 agent 交互、或者物理世界模拟。Harness 决定了模型如何跟系统其他部分交互。

对 coding agent，可以说 harness 就是 CLI coding agent，terminal 就是 environment。但这不是通用的——有时 system prompt 加一些 tools 就是 harness，有时会 spawn 子 agent，子 agent 也有自己的 harness。我们认为 harnesses 会不断进化，有 Cambrian explosion 式的各种使用模型的方式。

所以我们用 environment 作为抽象层——agent 太窄，harness 也太窄。任何类型的系统-模型交互都在 environment 的范畴里。

## RL 与 Post-training 的关系

**Sonya Huang**

是不是所有公司都应该用 environments 来 post-train 模型？有没有绝对需要用 environments 的场景，还是说可以用别的方法？

**Will Brown**

Environments 是工具，可以做所有这些事。它不把你绑死在 RL 上。比如小模型从大模型蒸馏——你把大模型放进 environment 跑很多次，得到同一交互协议的大量数据，用同一个 grader 筛选最好的例子做 SFT。你还可以做 prompt optimization、AB test 不同模型。

我们把 eval 看作一种用于衡量性能但不用于训练的 environment。很多人做 eval 就是在衡量当前性能然后迭代。RL 就是把这种迭代规模化——自动化地改一点模型、改一点 prompt，然后 hill climb。

**Sonya Huang**

你把 reinforcement learning 跟 post-training 看成同义词吗？

**Will Brown**

RL 是现在的主流，在大规模模型上你会花最多时间和算力在 RL 上。但它不是唯一要做的事——还包括 prompt tuning、SFT、在线 distillation。有很多算法都属于这个范畴，RL 是中间最大的一块，但周围还有很多东西。让用户有各种旋钮可以调，才是解锁的方式。

## 从游戏入门到复杂环境

**Sonya Huang**

有没有客户是拿你们做那种老派的 Atari 式学习的？

**Will Brown**

我们聚焦在 LLM 和类似 LLM 的 foundation model 上。但确实有很多研究人员用我们的平台做更像游戏的例子——比如 New York Times 的 Wordle，它成了很好的 hello world environment。基础设施简单但很有表现力，你会看到模型在学习如何思考这个游戏，给它奖励它就会做得更好。

它在难度上正好处在 sweet spot——一两个 GPU 上跑一个小时，就能看到模型真正学会怎么做得更好。

**Johannes Hagemann**

那些玩具般的游戏例子通常是入门途径。我们还有 RL residency 项目，一开始 8 到 10 人，现在有 14 到 16 人了。有研究生也有全职或兼职的人在做 RL environment 并做前沿研究。他们构建了很棒的 environment——从可验证的软件工程、Lean、医疗物理环境到网络安全环境。

## 构建高质量 Environment 的挑战

**Sonya Huang**

我听说一些 foundation model 公司花了数百万美元在每个这样的 environment 上。你提到了网络安全——构建一个网络安全 environment 需要什么？

**Will Brown**

网络安全领域有很多 capture the flag 游戏——系统有隐藏的 bug，本来是给程序员 hackathon 用的。这些挑战可以适配给 LLM，变成一个完整的软件环境：agent 在 terminal 里，有 bash 工具，可以跟文件交互。然后标记 rollout 完成，用其他代码来评估状态。

很多人用 RL 来评估数据质量——在一组任务、一组 rubric 上训练模型，可以发现 environment 里的 bug。因为 RL 中如果 environment 有 backdoor，模型可以钻空子。在 environment 上做小规模 run 可以帮助发现这些问题。

**Sonya Huang**

人们构建的 toy example 能否真正扩展到真实世界的复杂性？有没有类似机器人领域的 sim-to-real gap？

**Will Brown**

复杂性上没有那么大障碍——任何在计算机或网络上的东西都可以是 environment。真正的瓶颈往往是模拟的成本。关键是找聪明的方法来 mock 系统里合适的部分，而不是每次都要完整的东西。

就像地图游戏——有些部分是暗的，你走过去才照亮。如果你知道哪些部分会被探索，就可以决定哪些重要。比如 Tattoo Bench，关于客服 agent 涉及数据库——如果知道 agent 只会问某些类型的问题，你不需要几百万条记录的完整数据库，一个 mock 数据库就够了。

**Sonya Huang**

那你们的平台在这方面有帮助吗？如果创建 environments 是瓶颈，那么高效创建 environments 就是平台的核心能力了。

**Will Brown**

是的，这是我们的设计核心。我们在 coding agent 方面深入了很多细节——沙箱、终端状态、快照、跟不同 agent harness 交互的协议。复杂性会无限增长，所以我们尽量保持开放，有基础层的通用 environments，然后按需分支。有些已经建好，有些准备好等需要时再建。

我们也考虑如何做好的开发者体验——文档、agent skill files。做这类研究时会产生大量 institutional knowledge，我们把它设计成可以随着研究和最佳实践的演进而不断进化的东西。

## Environment 与数据标注时代

**Sonya Huang**

你们读过 Sutton 的 The Bitter Lesson 吧？经验的年代，Scale AI 时代的数据标注。你们觉得构建 environments 是不是那个时代的自然继承者？

**Will Brown**

看起来已经就是了。主要实验室的焦点已经转移了，但人类数据不会消失——创建 environment 本身就是针对模型还不够好的事情。识别哪些信息人类可以最独特地帮助模型提升技能是关键。

现在 RL 是最有效的方式：有任务、有 prompt、用 LM judge 评分、有成功 rubrics。最直接的形式是一组 yes/no 问题——LM 的答案里有没有做到这个？然后转成 reward score。这仍然需要大量人类数据。

## Environment Hub 的社区与生态

**Sonya Huang**

为什么创建 environment hub？

**Johannes Hagemann**

之前很多开源 repo 里挤满了各种 environment 实现。在开始 hub 之前我们已经创建了 verifiers framework，它本身就是标准化整个过程的方法。更重要的是，hub 可以围绕 environments 构建大量基础设施——这是单纯上传到 GitHub repo 做不到的。

我们有完善的 evaluations 集成在 environments 里，可以立即在所有前沿模型上测试。你可以轻松地在不同 trainer 里安装这些 environments。我们有自己的大规模 trainer Primer RL，但我们对开源很开放，可以集成多种不同的 trainer。

**Will Brown**

很多合作方把 environment 保持私有，这可以理解。但 hub 的价值在于，他们可以用已知的 environment 做 ablation，把私有的跟公开的做比较。或者做 eval 时，有统一实现的主流 benchmark 可以插入 training run 来监控进度。

**Sonya Huang**

现在 hub 上最受欢迎的 environments 是什么？

**Will Brown**

最受欢迎的往往是用作文档例子的那些。Wordle 是一个，但最受欢迎、大家 fork 和修改最多的叫 Wiki Search——在 Wikipedia 页面上搜索，被设计成可以广泛用于 agentic search 的模板。你只需要替换文档，其他都已设置好。

## RL 的效率问题与未来方向

**Sonya Huang**

Andrej Karpathy 是你们的投资人之一。他有一句名言——RL 很棒但很低效，就像用吸管吸比特。你们同意吗？在研究方面，怎么让 RL 更高效？

**Will Brown**

RL 确实用了大量算力来获取相当小的信息信号，但在某种程度上那也是它的价值所在。很多实验室聚焦 RL 是因为高质量人类数据难以扩展，而 RL 允许你用算力换数据——用更多算力从更少数据中获得很大价值。

在你不一定有 golden example 的情况下 RL 也很有用。如果你已经到了能访问的最大模型，就需要进入未知领域——探索是 RL 的核心。也许有人会发现比 RL 更高效的探索方法，但这就是目前用算力提升能力的前沿。

**Johannes Hagemann**

我不能替 Andrej 说话，但他在 Dwarkesh 播客后的看法可能会变化——特别是 Claude Code 在过去一个月变得更火了，他关于 RL 在 coding 领域有用性的看法可能会调整。不过我们不认为 RL 就是终点，在推动 agentic AI 能力方面还有很多低垂果实。

一个明显的限制是 context 方面——现在 context 能放的 token 有硬上限，我们一直在思考如何改进。

## Open-weight 模型与定制化

**Sonya Huang**

Open-weight 模型扮演什么角色？你们的基础设施只工作在 open-weight 模型上吗？能不能帮人用 closed-weight 模型做 post-training？

**Will Brown**

Trainer 本身需要访问权重，但 environment 的概念对各种优化都通用。我们可以在 environment 层面对 closed 模型做 eval、prompt tuning、model selection、评估 agent harnesses。不管 open 还是 closed，你都可以用 environment 作为数据引擎来蒸馏。

**Sonya Huang**

那你能不能做 LoRA？

**Will Brown**

LoRA 是 fine-tuning 的一部分，我们也推荐很多人用 LoRA 做 RL。你不需要完整权重，但不能为 GPT-5 上传自己的 LoRA。但我可以把 environment 带到平台上。需要用 LoRA adapter 还是 full fine-tuning，取决于 training recipe 和优化目标。

**Sonya Huang**

那你能不能在一个 closed 模型的 agent harness 上做 reinforcement learning？

**Will Brown**

这属于 prompt optimization 的领域——有人在做研究，某种程度上是 RL 在 prompt 空间的类比。比如 DSPy，GEPA 算法去年也得到很多关注。我们的 environments 也支持这个——你可以优化工具的 prompt、agent skill、system prompt。

## Recursive Language Models 与前沿研究

**Sonya Huang**

说到 DSPy，他们还有一个新东西——Recursive Language Models。你们怎么看？

**Johannes Hagemann**

我们对 longer horizon agent 很感兴趣。现在人们建了很多 scaffold 做 context management，但我们相信更有趣的方向是让模型自己学会管理自己的 context。RLM（Recursive Language Model）方向是我们认为最有前途的方向之一，Alex Sherstinsky 是原作者。

基本思路是给语言模型一个 Python wrapper 里的持久变量，它不用把整个 context 作为输入，而是通过这个变量检索、转换、管理自己的 context，还可以调用子元素——这就是 recursive 的由来。它已经在很长的推理 benchmark 上显示显著提升。我们接下来几个月会做在其中的训练——让模型学会真正使用这个 harness。

**Sonya Huang**

研究领域还有什么？你们觉得下一个前沿是什么？

**Will Brown**

我对 synthetic data 研究特别兴奋。感觉有很多我们应该能做到但还没看到的东西——创造性的 self-reflection 方式。很多人谈 continual learning，把我们已经知道的技巧用不同方式组合起来——prompt optimization、distillation——跟 synthetic data 结合。

感觉有很大探索空间——让模型自己筛选训练数据、甚至自己筛选 environments，理解哪些方式对 lifelong learning 最有效。
