构建 RL Environments 的 GitHub：Prime Intellect 的 Will Brown 与 Johannes Hagemann

本期节目邀请到 Prime Intellect 的联合创始人 Will Brown 和 Johannes Hagemann，探讨如何将前沿 AI 实验室的 post-training 和 reinforcement learning 基础设施民主化。Johannes 介绍了他们的全栈平台 Lab 和 environment 概念——将 RL environment 与 eval 统一为同一抽象，让任何公司都可以构建自己的产品-模型优化循环。他们讨论了 Cursor 如何通过将产品本身作为 RL environment 来优化模型，分析了 agent harness 与 environment 的关系，以及何时应该构建自定义 environment 来训练模型。Will 分享了他们与 RCI 等实验室的合作经验，Johannes 则介绍了 environment hub 的社区生态、Wordle 和 Wiki Search 等流行环境、网络安全环境的 sim-to-real 挑战，以及他们如何看待 RL 的效率问题与 Recursive Language Models 等未来研究方向。两人最后以「每家公司都是 AI 实验室」的愿景收尾，强调 institutional knowledge 的积累和模型定制化的重要性。

真正的瓶颈不是技术，是专长

Will Brown

真正的专长才是瓶颈所在。你更希望历史上最聪明的人为你工作，还是那个在公司干了三十年的人？有时候，你真正需要的是那个深耕了三十年的人。大量的 expertise 来自对一个问题的深度理解和长期互动，这正是 training 过程中发生的事情——几乎无法通过一段简短的 prompt 来复制。你真正需要的是让 institutional knowledge 随时间积累，让 best practices 随时间沉淀。这就是机构和公司变得真正强大和成功的方式——它们站在自己过去积累的基础上，而不是每天都推倒重来。

Prime Intellect：让前沿实验室的基础设施民主化

Training Data Host

Prime Intellect 现在是 AI 领域最酷的新实验室之一。你们的使命是让前沿实验室的训练能力对所有人都可及，这是一个非常崇高的使命。你们推出的 reinforcement learning environments hub 非常独特，大家都很兴奋。今天想聊的话题很多：post-training、reinforcement learning、agent harnesses、你们的平台、RL hub，还有关于 post-training 和 RL 未来发展的大问题。

Johannes Hagemann

我先从较高层面介绍一下我们的平台和研究。我们努力让前沿基础设施对任何 startup、企业以及新的实验室都可及。这些基础设施现在基本上被大实验室锁在墙内，没有人能真正接触到。我们从 compute 层和 compute orchestration 层开始，一路向上覆盖到整个完整的 post-training stack，包括大规模 reinforcement learning 所需的 training frameworks、社区方式运作的环境 hub，以及安全代码执行的 sandbox 和环境 hub 自带的 evaluations。

为什么要追求这个使命

Johannes Hagemann

原因有很多。一个是我们非常热衷的开放科学——历史上重大的科学发现都是全球共同讨论、共同推进的，大家可以在此基础上继续构建。但更实际地说，model customization 也有巨大的价值。那些成功的应用都是用 AI 做某件特定的事情——某个 agent、某个 workflow——同时还需要规模化交付并保持成本效益。要真正端到端地优化这些系统，你需要直接访问模型权重，为你的问题定制最好的模型，而不是在 prompt 层面做调整。

每个公司都会成为 AI 公司

Johannes Hagemann

我们坚信每个公司都会成为 AI 公司，而且大多数 AI 公司都会想拥有自己的 AI research lab。Research 可以有多种形式——可以是 pre-training，特别是在不只是 text in、text out 的领域。但在很多情况下，它会是 post-training——agentic 或其他类型的、针对特定任务和 workflow 的专注模型。我认为这已经达到了可产品化和成本效益的临界点，让人们能够大规模地做这件事。

Lab 平台与 environment 的概念

Johannes Hagemann

我们有一个全栈研究平台叫 Lab。Lab 的目标是让每个人都有能力做前沿研究实验室内部能做的事情。Lab 的一个核心重点就是 environment 这个概念。RL environment 封装了做 reinforcement learning 所需的一切，让模型可以通过试错来改进。但它不止于 RL。如果你以前没听说过 environment，它基本上就跟人们谈论新模型发布时报告的 evals 是一回事。SWE-bench、AMIE、Terminal Bench 都是 environment 的例子——它们有一组任务数据集，有一个供模型使用的 harness，还有一个负责对输出质量评分的 rubric 或 reward function。你在离线评估中用作 test set 的东西，完全可以在 reinforcement learning 中用作 training set。

我们的平台让用户能在 workflow 中使用 environments 来做 post-training、evaluation、synthetic data、reinforcement learning。它也是一个像 GitHub 一样的社区平台——这些东西是新的、复杂的，我们希望人们有大量可用的构建模块和示例，也希望大家可以协作、分享。

产品-模型优化循环：Cursor 的启示

Johannes Hagemann

总体思路是让更多公司目前只有大实验室才有的产品-模型优化循环。ChatGPT 是 OpenAI 创造的，Claude Code 是 Anthropic 创造的——它们有能力针对自己的特定 scaffold 优化模型，所以这些模型在自己的产品中运行得更好。但大实验室自然没有动力让模型在其他编程 startup 上运行得更好。Cursor 是一个很好的例子——它们构建了自己的 composer 模型，做了大规模 post-training，优化的 environment 其实就是 Cursor 本身。我们相信会有更多 startup 朝这个方向走，既优化现有产品，也构建全新的产品。

Environments 就是 Evals

Training Data Host

我想追问一下你说 environments 其实就是 evals 这个观点。在我理解里，environment 是 state，是对世界状态的描述——你观察，采取行动，观察世界状态的变化，然后更新 world model。这在我看来和 eval 是不同的。你能帮我弥合一下这两个概念吗？

Johannes Hagemann

在一两年前，eval 更多是问答形式的——一大题库。而 environment 的概念，人们想到的是老式的 RL，比如 Atari——那种长时间的状态交互循环。但现在两者已经合二为一了。特别是那些你想用来做大规模训练的 environments，它们有复杂的状态，可能是在模拟一个 web app，是一个完整的编码平台。但在原始的 RL 游戏中一直有一个 reward、一个 goal，有办法衡量进展和性能。这就是我所说的——存在一个目标，有办法衡量表现好坏。

Harness 与 Environment 的关系

Johannes Hagemann

Harness 是 environment 的一部分。对于任何 eval 或 environment 任务，有输入，一系列操作，然后有输出状态被评分。这整个中间的部分就是 environment，而 harness 定义了模型如何与系统的其他部分交互。对于 coding agents，CLI coding agent 是 harness，terminal 是 environment。但有些情况下 system prompt 加一些工具就是 harness；有些情况下会产生子 agent，子 agent 也有自己的 harness。我们用 environment 这个抽象概念，因为 agent 太窄，harness 也太窄——但 environment 作为一个整体抽象，任何系统-模型交互都在范围内。

什么时候应该构建 RL Environment

Johannes Hagemann

我觉得两者都有适用场景。如果你要训练一个非常擅长写 Rust 代码的模型，你可能希望它对很多不同应用场景都好用，所以你会用不同的 environments。还有一种情况是产品本身就是面向终端用户的界面，用户直接和 agent 交互，这种情况下最合理的做法可能就是让产品直接变成 environment。那些在意用 Claude 还是 GPT-5 还是 Gemini、在意能否选择模型的公司，它们已经在构建的基础设施和做 RL 所需的基础设施是同一套。所以 RL 是一个非常有用的工具，让人们在系统优化工具箱里多一个选项。

Environments 可以做更多事情而不只是 RL——比如从大模型蒸馏：把大模型放进 environment 运行很多次，然后用 grader 过滤出最好的例子做 SFT fine-tuning。也可以用 environment 做 prompt optimization、不同模型的 AB test。每个 AI 公司都应该优化他们的 AI 系统。

RL 与 Post-Training 的关系

Training Data Host

你认为 reinforcement learning 和 post-training 在功能上是同义词吗？

Johannes Hagemann

RL 是大事，如果你在做大规模模型，RL 会是你花最多时间、精力和算力的地方，但它不是唯一要做的事。整个 process 包括 prompt tuning、SFT、online distillation 等等。很多算法都归在这个大伞下——RL 是中间的大块头，但周围还有很多东西。

目标客户：Fortune 500 的 AI 工程师们

Johannes Hagemann

大多数公司都有能做这件事的人。任何财富 500 强公司很可能都有一支 AI 工程师团队——跟得上最新工具，擅长使用 Claude Code，对模型和 prompt 有很多见解。他们就是我们眼中的目标客户。当然，特别是给他们合适的工具之后。有些大公司里可能没有人能调试 TPU 集群来启动这样的运行，所以我们把整个基础设施的复杂性抽象掉，让他们不用操心。

客户故事：RCI 与医疗 AI

Johannes Hagemann

有一个我们最喜欢的客户 RCI，是一家专注于前沿开放模型的新实验室。他们一直在和我们合作，覆盖了整个技术栈。我们和他们一起训练了一些最大的 mixture of expert 模型，并且开源了。在 post-training 方面也和他们有合作。

Will Brown

他们是我们的密切合作伙伴。他们有很多需要基础设施的地方，这推动我们构建了很多组件——从 compute orchestration 到 post-training、pre-training、inference。他们在开放使命上和我们非常一致，但更偏向企业和终端用户。我们则专注基础设施层的开发者体验，让从想法到可部署模型的过程尽可能顺畅高效。

Johannes Hagemann

在研究社区方面，很多研究生和早期从业者在使用我们平台。也有很多专注于特定领域的实验室，比如医疗 AI。像 OpenMed 这样的团队想创建 benchmark 来了解模型在医疗方面的表现，他们真正关心的是赢得医疗专业人士的信任。对他们来说，拥有一个平台来创建 evaluation、展示它们、用它们来改进模型能力，再部署到本地医院——这种定制化能力至关重要。

Wordle、游戏与 Residency 项目

Johannes Hagemann

一个有趣的例子是《纽约时报》的 Wordle 游戏，它成了一个很好的 hello world environment——基础设施很简单但表达力很强。你可以在几块 GPU 上一小时内看到模型学会怎么玩得更好。那些更像玩具的游戏例子是人们学习如何构建 RL environments 的起点。另一个群体是我们的 RL residency 项目参与者，他们在 hub 上构建更复杂的环境，覆盖从可验证的软件工程到医疗物理、网络安全等各个领域。

网络安全环境与 Sim-to-Real

Training Data Host

我听说有些基础模型公司花了几百万美元在每个 environments 上。构建一个网络安全 environment 需要什么？人们构建的东西能真正扩展并反映现实世界的复杂性吗？

Johannes Hagemann

原则上它们可以做到你想要的任何复杂度。任何在计算机或计算机网络上的东西都可以成为 environment。真正的瓶颈是模拟器的成本。很多精力花在找巧妙的方法来 mock 系统的正确部分。比如你想让 agent 搜索整个互联网，有时需要真正的网络搜索，有时可以设计不需要完整搜索引擎的任务。有一个 benchmark 叫 Tattoo Bench，涉及客服 agent 和数据库——在真实系统中你需要完整数据库，但如果你知道 agent 只会问特定类型的问题，你就可以创建一个更便宜的内存 mock 数据库。

我们围绕这一块做了很多工作。以 coding agents 为例——sandbox、terminal states、快照功能、与不同 agent harness 交互的协议——这些都带来了大量复杂性。我们在设计时尽量保持开放，留出空间来构建新功能。从基础层的通用 environments 开始，你可以继续深入：coding agent environment 带全局 sandbox，或者每个 rollout 一个 sandbox。我们也花了大量心思在开发者体验上——文档、agent 的 skill files。

The Age of Experience：

Training Data Host

你们觉得构建 environments 是「The Age of Experience」的自然继承者吗？

Johannes Hagemann

我觉得基本上已经是了。大实验室的重心已经转移。他们仍在使用大量人类数据——人类数据不会消失，因为创建 environments 就是针对模型还不太擅长的事情。关键是识别出人类最能独特地帮助模型提高技能的信息。目前 RL 是最有效的方式——有任务、有 prompt、有 LM judge 评分、有 rubric 定义成功的样子。有时候通过 reward model，但最直接的版本就是一组是非题，LM 答案里有没有做到这些，这就变成了 reward score。

为什么要创建 Environment Hub

Johannes Hagemann

环境中心的想法源于我们看到很多开源仓库里有各种环境的混乱实现。在推出环境中心之前我们已有一个很好的验证器框架。除了共享环境、拥有一个开源平台之外，关键还在于有一个地方可以围绕它们构建大量基础设施，这是上传到 GitHub 做不到的。拥有适当的评估系统与环境集成，可以立即在所有前沿模型上测试，这是人们大量使用环境中心的原因之一。

Training Data Host

社区行为是什么样的？你看到有人在复刻、修改这些环境吗？

Johannes Hagemann

确实有很多合作者希望保持私有，这可以理解。但拥有中心的优势在于——可以在已知环境上做消融实验，将自己的私有环境与公开的同类任务环境对比。对于评估来说，有统一实现的流行基准测试也很有价值。最流行的环境往往是文档中的示例——Wordle 和 Wiki Search 是受欢迎的例子。Wiki Search 是在维基百科上做搜索，但被设计成可更广泛用于 agentic 搜索的模板。

RL 的效率问题与上下文限制

Training Data Host

Andrej Karpathy 也是你们的投资人之一。他有一句名言：RL 很棒但效率很低，就像用吸管吸数据流。你们同意吗？

Johannes Hagemann

RL 确实用了大量计算获取相对较小的信息信号，这是事实。但某种程度上这也是它的价值所在。很多实验室聚焦 RL 是因为人类数据是难以扩展的瓶颈，尤其是高质量的人类数据。RL 允许你用计算换取数据——用更多计算从更少数据中获取大量价值。没有 golden examples 时它也很有效。也许人们会发现比 RL 更高效的探索方法，但这就是目前用计算来探索和提升能力的前沿。

看看 Claude Code——它之前已经很受欢迎，但在过去一个月左右更加火爆了。总的来说，我们认为这不会是终点。在推动 agentic AI 能力方面还有很多低挂的果实。但我们也很清楚目前的局限性——就是上下文方面。目前可塞进一个上下文窗口的 token 数量有限，我们一直在思考如何改进这一点。

开放权重与封闭权重模型

Training Data Host

你们的基础设施是在开放权重模型上最优工作，还是也能帮助人们在封闭权重模型上做 post-training？

Johannes Hagemann

训练器本身需要访问权重。但 environment 的概念是通用的，我们可以在 environment 层面用相同的基础设施对封闭模型做 eval、prompt tuning、模型选择、评估 agent harness。无论是使用开放还是封闭模型，或者把 environment 当作一个数据引擎，有很多方法可以用这些工具来优化模型。LoRA 属于 fine-tuning 过程的一部分，你不需要完整权重。但在很多情况下，你需要 LoRA adapter 还是完整 fine-tuning，取决于训练方案和优化目标。

Recursive Language Models 与长 Horizon Agent

Johannes Hagemann

我们对长 horizon agent 很感兴趣。我们内部已思考了很久如何让模型学会管理自己的上下文。Recursive language model 的研究方向是我们的重点之一。从 Alex 发表 RLM 工作以来我们就一直非常感兴趣。几周前有博客文章展示了使用 RLM harness，给语言模型访问一个持久 Python 包装器中的变量，让它检索、转换和自行管理上下文。这被证明在长 horizon reasoning 基准测试上有显著提升。我们非常兴奋——真正训练模型使用这个 harness，将是接下来几个月的工作方向。

Synthetic Data 与 Continual Learning

Johannes Hagemann

我对 synthetic data 研究非常兴奋。感觉有很多事我们理论上应该能做，但还没在公开领域看到——比如自我反思的创造性方式。大家经常讨论 continual learning，模型自我学习的能力会越来越重要。将已知的技巧以不同方式组合——prompt optimization、distillation、synthetic data——感觉还有大量探索空间。让模型策划自己的训练数据，甚至策划自己的 environments，以及理解哪些版本对终身学习最有效。

乐观的未来：每家公司都是 AI Lab

Training Data Host

如果一切顺利，世界会是什么样子？Prime Intellect 在那个世界中扮演什么角色？

Johannes Hagemann

我们不希望未来 AI 在所有垂直领域的价值被大实验室垄断。我们希望赋能创业者和企业，让他们不被碾压，更好地优化产品。我们希望实现更多像 Claude Code 那样的时刻，更多「Cursor-for-X」那样的时刻。每家公司都是一个 AI lab——某种程度上，是的。

Will Brown

有时候你真正需要的是那个在公司干了三十年的人。大量的 expertise 来自对一个问题的深度理解和长期互动，这正是 training 中发生的事情。你真正需要的是让 institutional knowledge 随时间积累，让 best practices 随时间沉淀。这就是机构和公司变得真正强大和成功的方式——它们站在过去积累的基础上，而不是每天推倒重来。我们希望任何想这样做的公司都能拥有这种能力。随着软件变得更容易操作，编程门槛越来越低，AI 研究也在发生同样的变化。

Training Data Host

这是一个非常鼓舞人心的愿景。非常感谢你们今天的参与。你们在 environments 和 environment hub 方面开辟了道路，感谢你们花时间帮大家解密 environment 这个概念。
