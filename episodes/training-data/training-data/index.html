构建 RL Environment 的 GitHub：Prime Intellect 的 Will Brown 与 Johannes Hagemann

本期节目采访了 Prime Intellect 的联合创始人 Will Brown 和 Johannes Hagemann。Prime Intellect 致力于将前沿 AI 训练基础设施开放给所有公司，打破大实验室对 post-training 能力的垄断。他们详细介绍了全栈研究平台 Lab 的核心概念「environment」——一种将评估（eval）与强化学习训练集统一起来的抽象层。节目深入探讨了环境（environment）与代理框架（agent harness）的关系、产品-模型优化循环（以 Cursor 为例）、如何构建网络安全等专业领域的环境、以及从概念到可部署模型的 pipeline。他们还分享了关于 recursive language model、合成数据、开源与闭源模型策略的前沿思考，并对 AI 行业未来做出了「每家公司都成为 AI 实验室」的乐观展望。

专业经验才是真正的瓶颈

Will Brown

真正的专业经验才是瓶颈。你是愿意让历史上最聪明的人为你工作，还是让一个在公司待了三十年的人？有时候你真正需要的是那个积累了三十年经验的人。很多专业经验来自于深入理解一个问题，并且在很长时间里持续与其互动。这在训练中尤其明显——这种积累几乎不可能通过一次简短的 prompt 来复现。你真正需要的是让机构知识随时间不断复利，让最佳实践不断积累。

这就是机构和公司变得真正强大和成功的方式——它们站在过去成果的肩膀上，而不是每天都从零开始。我们希望任何想这样做的公司都能获得这种能力。特别是随着软件变得越来越容易操作，编码的门槛越来越低，我们看到同样的事情也在 AI 研究中发生。

Prime Intellect：让前沿训练能力开放给所有人

Michael Bernstein

Prime Intellect 是现在 AI 领域最酷的新实验室之一。他们的使命是让前沿实验室的训练能力对所有人开放，这是个非常崇高的使命。他们有着非常好的品味和对开发者体验的理解。他们推出的 reinforcement learning environments hub 非常具有差异化，大家对此非常兴奋。

Johannes Hagemann

我们试图把前沿基础设施提供给任何初创公司、企业和新实验室。这些基础设施目前都被锁在大实验室的高墙之内，没有人真正能接触到。我们从计算层和计算编排层开始，一路往上做到完整的 post-training 全栈，包括大规模 RL 所需的训练框架、通过环境 hub 以社区方式构建的环境、用于安全代码执行的 sandbox，以及环境 hub 中的 evaluation 模块。我们把它作为一个端到端的产品来提供。

开放科学与模型定制化的双重驱动

Johannes Hagemann

原因有很多。一个原因是我们对开放科学充满热情——历史上很多重大的科学发现都是大家讨论后，全世界在此基础上继续发展的。从更实际的角度来说，模型定制化有巨大价值。那些成功的应用是针对特定场景、特定 agent、特定 workflow 来使用 AI 的。你还希望以规模化和成本效益的方式交付这些能力。

真正优化这些系统的端到端方式，就是能够直接访问模型权重。你可以把模型塑造成最适合你问题的模型，而不是用一个只能在 prompt 层面做调整的通用模型。这本质上是在 prompt 层面之上提供更深层的定制化能力。

每家公司都是 AI 公司

Will Brown

我们坚信每家公司都会成为 AI 公司，而且大部分 AI 公司都会想要有自己的 AI 研究实验室。研究可以有很多形式，可以是 pre-training，特别是如果你的领域不满足于 text-in-text-out 而是需要更定制化的东西。在很多情况下会是 post-training，无论 agentic 还是其他方向——针对特定任务和 workflow 的专注模型。这已经到了可以产品化、成本效益高的阶段。

Lab 平台与 Environment 的概念

Will Brown

我们有一个全栈研究平台叫 Lab，目标是让每个人都有能力做到前沿研究实验室内部能做到的事情。Lab 的一大重点就是「environment」这个概念。很多人可能在 reinforcement learning 的语境下听说过这个词，RL environment 封装了做 RL 所需的一切，让模型可以通过试错来改进。但它不仅仅局限于 RL，它更通用。

如果你之前没听说过 environment，它本质上等同于新模型发布时报告的那些 eval。比如 SWE-bench、AMIE、Terminal Bench，这些都是 environment 的例子。它们有一组任务数据集，有 agent 运行的 harness，还有一个叫 rubric 或 reward function 的东西负责对输出质量进行评分。你在离线场景中用作 eval 的东西，同样可以用在 RL 中作为训练集。这是一种交互式提升模型性能的方式。

我们的平台让用户可以在 workflow 中使用 environment——用于 post-training、evaluation、synthetic data、reinforcement learning。

它也非常注重社区平台的方向，就像 GitHub 那样。这些东西是新的、复杂的，我们希望人们有大量的构建模块可以借鉴，有大量的例子可以学习，希望它是协作性的。人们可以把使用模型或工具的方式做成 environment，让其他人在那些 environment 中 post-train 模型，这样就能以共享的方式在不同任务上提升性能。

产品与模型的优化循环

Johannes Hagemann

总体思路就是让更多公司拥有大实验室才有的产品-模型优化循环。它们可以针对自己的产品来优化模型。比如 ChatGPT 由 OpenAI 创造，Claude Code 由 Anthropic 创造，它们确实有能力针对自己的特定 scaffold 来优化模型。

随着这些产品越来越流行，大实验室反而越来越没有动力让模型在其他编码初创公司那里表现得更好。我们的想法就是给那些公司工具，让它们拥有自己的产品-模型优化循环。一个很好的例子是 Cursor——它们很早就意识到了这一点，构建了自己的 composer 模型，做了大规模的 post-training，而它们使用的 environment 就是 Cursor 本身。

Environment 与 Eval 的融合

Michael Bernstein

我们来拆解一下这个说法。在常规理解中，environment 是有状态的——它描述世界状态，你观察它，采取行动，观察世界状态如何变化，然后更新自己的 world model。这跟 eval 是有区别的——eval 更像是「你应该在这组问题上答对这个答案」。

Will Brown

eval 有一种版本是一两年前的状态——很多 eval 就是问答形式，一个很大的题库。而提到 environment 时会想到老派的 RL，比如 Atari 游戏那种长时间运行的状态交互循环。但现在这两者已经合二为一了。特别是你用来做大规模训练的那些 environment，它们确实有复杂的状态，可能在模拟一个 web 应用，也可能是一个完整的编码平台。关键在于有一个目标，不是让模型跑一遍流程然后让人凭感觉判断，而是确实有东西可以衡量进展和性能。

何时构建 RL Environment

Will Brown

两种都需要。比如说你要训练一个非常擅长 Rust 编码的模型，你希望它在很多不同的应用场景中都表现好，那就需要不同的 environment 来聚焦不同的领域任务。也许你是一家公司，想要一个模型非常擅长调用你的工具、使用你特定的领域语言。还有一种情况是你的产品本身就是面向终端用户的用户界面，用户直接与 agent 交互，这种情况下让产品直接变成 environment 可能最有意义。

那些关心自己用的是 Claude 还是 GPT-5 还是 Gemini，关心能否选择模型、评估 system prompt、更换 model endpoint、使用 mini 版本获得更好成本-性能平衡的公司，它们已经在构建做 RL 所需的基础设施了。最适合提升模型能力的训练范式跟很多人一直在搭建的系统优化基础设施其实是同一回事。RL 是一个非常有用的工具，人们可以把它作为系统优化的一个选项放在工具箱里。

Agent Harness 与 RL 的关系

Will Brown

harness 是 environment 的一部分。对于任何 eval 或 environment 任务，都有一组输入，然后一系列事情发生，最终产生输出状态然后被评分。整个中间环节——无论是与模拟器交互、与其他 agent 交互、还是物理世界模拟——这就是 environment。Harness 连接了模型与系统中其他部分的交互方式。

对于 coding agent，你可以说 harness 就是 CLI coding agent，而 terminal 就是 environment。但这不一定是通用的。在某些情况下，system prompt 加上一些 tools 就是 harness；在某些情况下，harness 会生成 sub-agent，而 sub-agent 又有自己的 harness。我们选择用 environment 作为抽象概念，因为 agent 和 harness 都太窄了。在 environment 里你可以做所有这些事情，任何系统-模型交互方式都在其范围之内。

Post-training 并非只有 RL

Will Brown

environment 是你可以用来做所有这些事情的工具。它作为一个抽象概念，不把你局限在 RL 上。比如我想有一个小模型从大模型蒸馏而来：你把大模型放到你的 environment 里让它跑很多次，然后用同一个 grader 过滤出最好的例子做 SFT fine-tuning。你也可以用 environment 做 prompt optimization 或 A/B 测试不同的模型。

RL 现在是重头戏，但 post-training 不是只有 RL。从初始模型到你想部署的系统，中间涉及很多环节——prompt tuning、SFT、online distillation，一系列算法都归在这个大伞下。RL 是中间最大的那个，但周围还有很多东西。

平台的用户群体

Will Brown

任何 Fortune 500 公司都可能有 AI 工程师团队，他们能跟上最新工具、善于使用 Claude Code，对模型和 prompt 有很多见解。这些人当然能做到，这就是我们眼中的目标客户群体。

Johannes Hagemann

尤其是如果你给他们合适的工具。有些大公司可能没有谁能调试 TPU 集群来真正启动这样的训练，或者缺少其他组件来让大规模 agentic RL 变得更容易。我们做的就是为他们抽象掉整个基础设施。

客户案例：RCI 与医疗 AI

Johannes Hagemann

RCI 是我们的重要客户，一个新实验室在做前沿开放模型。我们和他们一起训练了一些最大的 mixture of expert 模型并成功开源。

Will Brown

他们对基础设施有很多需求，推动我们构建了从计算编排到 post-training、pre-training、inference 的各个组件。他们在开放性使命上和我们一致，但更聚焦于企业和终端用户。我们的角色是专注于基础设施层的开发者体验，让从想法到可部署模型的过程尽可能无缝、快速、高效。

我们还和医疗 AI 领域有紧密合作。比如 OpenMed，他们的重点是赢得医疗专业人员的信任，所以他们真正关注的是领域本身。他们可以拥有这个平台来创建 evaluation、展示成果、提升模型能力，这些模型最终可以部署在医院本地。拥有定制化能力，对数据来源有端到端的信任和理解，对他们来说至关重要。

从 Wordle 到复杂环境

Will Brown

我们也有偏游戏方向的示例。一个经常用来做演示的例子是纽约时报的 Wordle 游戏，它成了一个非常好的 hello world environment。基础设施非常简单但表现力很强。你可以很快上手，然后就会看到一个「啊哈」时刻——当你给它奖励来做得更好时，模型开始学会思考这个游戏。用非常小的模型也能做到，你可以在几块 GPU 上运行一个小时，就能看到模型真的学会变得更好。

Johannes Hagemann

人们就是用这些玩具示例来学习如何构建 RL environment，这也是人们大量使用 environment hub 的原因。然后他们逐步构建更复杂的 environment。

强化学习驻留计划

Johannes Hagemann

我们的 RL residency 项目最初有八到十个人，现在大概有十四到十六人。有研究生，也有全职或兼职的人。他们在各个垂直领域构建了非常棒的环境——从可验证的软件工程、Lean、医疗物理环境，到一些网络安全环境。

构建网络安全环境

Will Brown

网络安全世界中有很多 capture the flag 游戏的工具——某个系统里有一个隐藏的 bug，这是为程序员设计的挑战。你可以把这些挑战移植到 LLM 上，它就成了一个完整的软件环境：一个 terminal，agent 生活在 terminal 里，可以访问 bash 命令的工具，使用 Claude Code 或其他 agent harness。然后某个时刻它标记 rollout 完成，你就可以用其他代码来评分环境的状态。

用 RL 评估数据质量

Will Brown

很多人有兴趣把 RL 作为评估数据质量的方式。通过衡量在一组任务、一组 rubric、一组 environment 上训练模型会发生什么，来发现 environment 中的 bug。如果你的 environment 有后门，模型可能会利用它玩弄系统。做小规模或中规模的运行可以让你真正探查问题可能在哪里。

环境构建中的 Sim-to-Real

Will Brown

在实际复杂性方面，障碍没那么大。任何在计算机上的东西都可以是一个 environment。真正成为瓶颈的往往是模拟器的成本。很多人在想办法巧妙地对系统进行 mock。比如有一个 benchmark 叫 Tattoo Bench 是关于客服 agent 涉及数据库的。在真实系统中你可能需要完整数据库，但如果你知道 agent 只会问特定类型的问题，你只需要一个 mock 数据库。识别出系统中哪些部分的复杂性是多余的，是任务设计过程的一部分。

平台的设计理念

Will Brown

我们在设计上努力保持开放性，为未来建设功能留出空间。有基础层的通用 environment，然后你可以深入不同的分支。我们预见到了会有很多这样的分支。我们还非常重视开发者体验——文档、agent 的 skill files，这些都可以随着研究文献的演进和最佳实践的清晰化而持续演进。

从《Age of Experience》到 Environment 构建

Will Brown

事实上，构建 environment 已经成为 Sutton 所说的 Age of Experience 的自然延续。很多大实验室的注意力已经转移了。人类数据不会消失，因为创建 environment 就是为了那些模型还不擅长的任务，这意味着人类在某些方面仍然比模型强。目前看来 RL 是最有效的实现方式——任务加上 prompt、用 LM 作为 judge 来评分、一个描述成功标准的 rubric。这需要大量的人类数据。

Environment Hub：标准化与社区

Johannes Hagemann

创建 hub 的想法源于我们看到很多不同的开源 repo 里各有不同的 environment 实现。我们之前已经创建了很好的 verifier 框架。更重要的是有一个地方可以围绕它们构建基础设施，这是上传到 GitHub repo 做不到的。你可以立即在所有前沿模型上测试它们，还可以在不同的 trainer 里轻松安装这些 environment。

Will Brown

对于 eval 来说，有一个统一实现的流行 benchmark 集合非常有价值。也许你在自己的 environment 上表现不错，但能不能泛化到其他任务上？有其他任务可用，可以帮助人们了解自己的任务之外，也知道希望模型在其他哪些方面也表现好。

最受欢迎的 Environment

Will Brown

最受关注、被 fork 和改出各种版本最多的一个是我们叫 Wiki Search 的 environment。它在 Wikipedia 页面上做搜索，但设计成可以用于更广泛的 agentic search 的模板。很多应用场景下，人们希望 agent 知道如何搜索内部文档。有了这个模板，你只需要替换文档，就有了一个可以直接使用的 environment。

RL 的效率问题与 Andrej Karpathy 的批评

Will Brown

RL 确实用了大量计算来获取很小的信号，但某种程度上这正是它的价值所在。人类数据——尤其是高质量人类数据——是一个难以扩展的瓶颈。RL 允许你用计算换取数据，从更少的数据中获取大量价值。另外，在你没有 golden example 的情况下，exploration 正是 RL 的核心——如何探索和尝试不同的东西。也许会有比 RL 更高效的 exploration 方法，但这目前就是前沿。

Johannes Hagemann

Claude Code 在过去一个月左右明显更加爆发了。Andrej 的观点可能会改变，关于 RL 在编码领域究竟有多有用。但总的来说，我们不认为这就是终点。我们想要始终处于下一个范式的前沿。

上下文窗口的硬性限制

Johannes Hagemann

我们也看到一些限制，主要是在 context 方面。当前能放进去的 token 数量有硬性限制，我们在考虑如何改进这一点。

开源与闭源模型

Will Brown

训练器本身需要访问权重。但 environment 的概念是通用的。我们可以在 environment 层面用同样的基础设施对 closed 模型做 eval、prompt tuning、模型选择、评估 agent harness。无论用 open 还是 closed 模型，你都可以把 environment 作为数据引擎来使用。

LoRA 我仍然认为是 fine-tuning 的一部分，这也是我们推荐很多人做 RL 的方式。我不能为 GPT-5 上传我自己的 LoRA，但我可以把 environment 带到平台上来。在 closed 模型的 agent harness 上做 RL 属于 prompt optimization 的范畴，这类似于在 prompt 空间里做 RL。

Recursive Language Model：模型自己管理上下文

Johannes Hagemann

我们对更长期 horizon 的 agent 非常感兴趣。我们内部一直在思考如何让模型学会管理自己的 context。recursive language model 是我们认为最有前景的方向之一。几周前有博客文章展示了使用 RLM harness，给语言模型访问一个持久 Python 包装器中的变量，这样它就不需要把整个 context 作为输入。它可以从变量中检索、转换、管理自己的 context，还可以递归地调用子元素。

已经证明在很长推理任务上能显著提升 benchmark 表现。实际在这个 harness 里训练模型，让模型学会使用它——这就是我们接下来几个月要做的。

合成数据与持续学习

Will Brown

我对 synthetic data 研究非常兴奋。把我们已经知道的技巧——prompt optimization、distillation——与 synthetic data 结合起来使用，这里面还有很大的探索空间。让模型管理自己的训练数据、管理自己的 environment，以及理解哪些版本对 lifelong learning 最有效，都还有很多空间。

愿景：每个公司都成为 AI 实验室

Johannes Hagemann

我们不希望未来的世界是 AI 在各个垂直领域的价值全部被大实验室垄断。我们希望赋能创业者和企业，让他们有工具来优化自己的产品。就是激发更多的「Claude Code 时刻」和「Cursor for X」时刻。

Will Brown

如果数据是瓶颈，真正的专业经验是瓶颈——你愿意让历史上最聪明的人为你工作，还是让一个在公司待了三十年的人？有时候你真正需要的是那个积累了三十年经验的人。这就是机构和公司变得真正强大和成功的方式——它们站在过去的肩膀上，而不是每天从零开始。随着软件变得越来越容易操作，我们看到同样的事情也在 AI 研究中发生。
