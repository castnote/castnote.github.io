Anthropic Boris Cherny：编码已解决，接下来是什么

这段现场访谈中，Claude Code 的创造者 Boris Cherny 分享了他对软件开发未来变革的洞察，讲述了产品从 2024 年底孵化器项目起步的偶然诞生过程。他阐述了“编码已被解决”的论断——自己已 100% 使用模型写代码，单日最高提交 150 个 PR。Boris 深入探讨了未来团队从工程通才到跨学科通才的转变，以及人人都会写代码的组织形态。他以印刷术类比软件开发民主化，分析了 AI 如何重塑商业竞争格局——Seven Powers 中哪些模式更重要、哪些被削弱。他还回答了关于产品与模型贡献比例、本地与云端 AI、MCP 与 Computer Use 等观众提问。

开场：Claude Code 的诞生

Asia

Asia 以一场现场互动开场：谁在用 Claude Code？谁有 Claude Code 精神病？她的团队半开玩笑说她自己也得了这个病。随后她介绍了今天的嘉宾——Claude Code 的创造者 Boris Cherny，说他坐拥第一排座位，亲眼目睹了现代软件开发方式被彻底重塑。

今天的采访由团队 Lauren Reader 主持。Asia 感谢 Boris 在打造整个产品的同时还抽空来和大家交流。

Boris 的身份与转变

Lauren Reader

Lauren 介绍了 Boris 的背景：除了创造 Claude Code，他是一个纯粹的工程师，写过《Programming TypeScript》，整个职业生涯写了大量代码。有趣的是，去年一整年 Boris 一行代码都没写过——至少 2026 年到现在是这样，这是巨大的转变。她还发现了一件很少人知道的事：Boris 初中时写过一篇 TI-83 Plus 计算器的 BASIC 编程教程，至今还在网上。

Claude Code 的起源

Boris Cherny

Claude Code 某种意义上是他偶然做出来的。2024 年底他加入了一个叫 Anthropic Labs 的孵化器团队，只有几个人，做出了 Claude Code、MCP 和桌面应用。团队后来解散，现在又重新集结准备做第二轮，由 Anthropic 首席产品官 Mike Krieger 领导。

开始做编码产品是因为他们感受到一种 product overhang——模型的能力已经远远超出了任何产品能捕获的水平。2024 年底编码领域最先进的还是 IDE 里的 type-ahead，一次补全一行代码，但他们知道可以走得更远。

头 6 个月很不顺利，基本将将能用，他自己大概只写 10% 的代码。真正的转折点是 Opus 4，指数增长从那时开始，每次模型发布都会再 inflection 一次。他们从一开始就清楚自己在做一个 pre-PMF 的产品，是为下一代模型打造的。

编码已解决

Boris Cherny

Boris 认为编码已经被解决。他从约 2025 年 10 月就开始 100% 用模型写代码，每天提交几十个 PR，最多一天做了 150 个。当然并非在所有地方都如此——庞大代码库或冷门语言还有挑战，但一切在快速进步。

Claude Code 本身的代码库是 TypeScript 加 React，选择这些是因为它们在模型训练分布上最充分。刚开始建库时模型没现在聪明，语言和框架的选择很重要。现在模型什么都能写了。

个人开发环境

Boris Cherny

Boris 的大部分工作是在手机上完成的。他装了 Claude app，左边有一个 code 标签页，通常同时有 5 到 10 个 session，每个 session 里又有好几个 agent，加起来几百个同时在跑。晚上通常有几千个在做更深度的工作。

他越来越常用的管理方式是 /loop，用 cron 调度 job 重复执行。他有几十个 loop 在 babysit PR、修 CI、自动 rebase、维护 CI 健康、抓 Twitter 反馈做聚类。Anthropic 刚推出的 routines 概念一样，但在 server 端跑，合上电脑也会继续。

未来团队：通才崛起

Boris Cherny

Boris 预测未来会有更多跨学科的 generalist——既擅长产品工程又擅长设计，或者产品、数据科学和工程都强。Claude Code 团队很多人就是这样的通才。团队每个人都会写代码：工程经理、产品经理、设计师、数据科学家、财务人员、用户研究员。

软件产品的未来与 SaaS 末日

Boris Cherny

Boris 用 Hamilton Helmer 的 Seven Powers 框架分析 AI 的影响。Switching cost 变得不那么重要，因为可以用模型把东西从一个平台搬到另一个平台。Process power 也在减弱。但 network effects、scale economies、cornered resources 仍然重要。

未来 10 年能够颠覆一切的 startup 数量会增长 10 倍。小 startup 可以做出跟大公司一样有价值的产品，因为大公司会遇到内部阻力，而创业者可以从零开始完全 AI-native。他认为这是最好的创业时机。

Q&A：产品还是模型

Dan

Dan 问 Boris：Claude Code 现在这么成功，你觉得有多少归功于底层模型的进步，有多少归功于你们的产品决策？

Boris Cherny

Boris 认为大概是 50/50。他参加过 YC，YC 反复强调 build something people love，产品是什么不重要，模型是什么也不重要，最终必须做出人们热爱的东西。这就是产品重要的原因——他们在每个细节上花了心思。

随着模型变好，harness 的重要性会下降。一年后模型的对齐会好得多，所有安全机制——prompt injection 防护、命令验证、permission mode、human in the loop——都会变得不那么重要。模型自己就会做正确的事。

编程的民主化

Audience Member

Claude Code 在文化上做了重要改变：它民主化了软件开发。店主可以给自己写软件，有人给微控制器编程控制灯光。他问未来写软件会不会变成像用 Office 一样的技能，每个人都能做。

Boris Cherny

Boris 非常肯定，甚至觉得还不止。他以印刷术类比：印刷术出现前欧洲只有约 10% 的人口识字。印刷术出现后 50 年内，出版的文献比之前一千年还多，书价下降 100 倍。几百年后全球识字率上升到约 70%，现在不需要读写学位就能读写。

即将发生的事情会比 50 年快得多：软件会完全民主化。领域知识才是难点，coding 反而是简单的部分。最适合写会计软件的人可能不是工程师，而是一个真正优秀的会计师。

内部差距与组织结构

Lauren Reader

Claude Code 公开发布前一直是内部工具。Lauren 问工程和外界差距有多大，差距在拉大还是缩小。

Boris Cherny

模型层面没什么差距，他们用的就是大家都能用到的模型。真正的差距在组织结构和流程。Anthropic 所有人所有事情都用 Claude，Claude 们通过 Slack 交流。公司已经没有手写代码，所有 SQL 都是模型写的。

并行化与 Loop

Jared

Jared 追问在模型和 harness 层面，怎么注入 priors、怎么让委派工作和并行化的体验更好。

Boris Cherny

随着模型变好，它会自然地做并行。比如 4.7 自己就会主动用 loop——你说去拉一下这个数据，它会发现数据在变化，主动开一个 loop 每 30 分钟发报告。如果还要用户来操心怎么更好地用工具，那是产品设计的问题。

云端还是本地

Audience Member

很多人用 Claude 或 Codex 在云端做计算，但也有声音支持本地跑 AI。未来几年大家会依赖云端集中式计算，还是转向本地 agent？

Boris Cherny

最根本的回答是这不重要。几年后模型会自己写所有代码、自己启动 agent、自己搭建环境。如果它决定用本地模型，那就用本地模型。这些不会再是工程师需要做的决策了。

知识工作与 MCP

Jamie Nester

Claude Code 利用了开发者工具大多在本地的优势，但知识工作有很多云端工具。怎么给 Co-work 足够的工具访问权限，让它像 Claude Code 对开发者那样强大？

Boris Cherny

对知识工作来说，答案一直是 MCP。Claude AI 里用的同一个 MCP connector——接 Salesforce、Google Docs、Google Calendar——所有产品都能用。Computer use 是一个 catch-all，速度还比较慢但做得很准。但对模型来说，一切只是 token。

给创业者的建议

Ryan

Boris 之前看到了 product overhang，做了一个会随模型变好而更有趣的产品。Ryan 问他今天会做什么样的产品——6 到 12 个月后随着模型变好而变得更有趣的产品？

Boris Cherny

Claude Design 就是一个很好的例子，现在已经很好用了，还会变得更好。Claude Code 的 loop、batch、大规模并行化 agent 也会继续进化。接下来几周会有新东西发布。
