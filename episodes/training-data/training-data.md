## Prime Intellect：让前沿 AI 训练基础设施人人可用

**Sonya Huang**

Will 和 Johannes 来自 Prime Intellect，这是目前 AI 领域最酷的新实验室之一。他们的使命是让前沿实验室级别的 training 基础设施对所有人开放。他们推出的 reinforcement learning environment hub 非常有差异化，获得了广泛关注。

**Johannes Hagemann**

我们致力于把前沿基础设施开放给所有的 startup、企业和新实验室。目前这些基础设施被锁在大实验室的围墙里面，别人根本用不到。我们从 compute 层和 compute orchestration 层做起，一路往上到整个 post-training 技术栈——包括大规模 reinforcement learning 的 training 框架、community 化的 environment hub、安全代码执行的 sandbox、以及 evaluation 功能。

## 开源科学与模型定制

**Sonya Huang**

你们为什么要追求这个使命？让所有基础设施对所有人开放，背后的直觉是什么？

**Will Brown**

原因很多。首先是 open science。人类进步的方式就是大家讨论和分享，在彼此基础上继续构建。但从更实际的角度来说，model customization 有很大价值。赢家级别的应用都是把 AI 用在特定场景——某个 agent、某个 workflow，而且需要规模化交付和高性价比。所以要端到端优化这些系统，你需要直接拿到 model weights，把模型打造成最适合你问题的版本，而不是只在 prompt 层面做定制。

## 每家公司都会成为 AI 公司

**Will Brown**

我们坚信每家公司都会成为 AI 公司，大部分 AI 公司都会想拥有自己的 AI research lab。Research 可以是 pre-training、post-training，做 agentic 或面向特定任务和 workflow 的模型。这已经到了可以产品化、规模化、成本可控的阶段了。

## 全栈研究平台 Lab

**Sonya Huang**

能从高层介绍一下你们的平台是做什么的吗？

**Will Brown**

我们有一个全栈研究平台叫 Lab。Lab 的核心理念是让世界上任何想做这类研究的人都拥有前沿研究实验室内部才有的能力。Lab 的重要焦点是 environment 概念。RL environment 封装了做 reinforcement learning 所需的一切。但它比 RL 更通用——本质上跟大家讨论新模型发布时提到的 eval 是一回事。SWE-bench、AMIE、Terminal Bench 都是 environment 的例子：一组任务数据集、一个让模型运行的 harness，还有一个 rubric 或 reward function 来评分。

你用作 eval 的那套东西在 RL 里就可以当 training set。我们的平台让大家在自己的工作流程中使用 environment——做 post-training、evaluation、synthetic data 或 RL。同时它注重社区属性，就像 GitHub 一样。这些东西新且复杂，我们希望大家有现成的模块和例子可以借鉴、协作和分享。

## 产品-模型优化闭环

**Johannes Hagemann**

核心理念是给更多公司提供目前只有大实验室才有的产品-模型优化闭环。ChatGPT 是 OpenAI 做的，Claude Code 是 Anthropic 做的——因为他们有能力针对自己的产品优化模型。随着这些产品越来越火，大实验室自然没什么动力帮其他 startup 把模型调得更好。所以我们要给他们工具来拥有自己的优化闭环。

一个很好的例子是 Cursor。他们很早就意识到了这一点，做了自己的 Composer One 模型，做了大规模 post-training。他们优化模型用的 environment 就是 Cursor 本身。我们相信会有越来越多的 startup 走这个方向，不仅优化现有产品，还会构建没有产品-模型优化闭环就不可能做出来的全新产品。

## Environment 与 Eval 的关系

**Sonya Huang**

你说 environment 其实就是 eval。能拆解一下吗？在我脑子里，environment 是对 world state 的描述，你通过它观察 action 和状态变化，更新 world model。这跟 eval——「在这组问题上你应该得到这个答案」——在我看来是不一样的。能帮我把这两个概念统一起来吗？

**Will Brown**

一两年前的 eval 基本是问答题。而老派 RL 的 environment 更像 Atari 那种长时间运行的状态交互循环。现在这两者已经融合了。做大规模 training 的 environment 有复杂状态，可能在模拟 web app 或完整的 coding 平台。但就像 RL 游戏一样，总是有 reward、有目标。关键是有一个东西能衡量进度和表现，不是让人来 vibe check。所以 environment 和 eval 在我们看来基本是同一个东西。

## 何时构建 RL Environment，何时用产品本身

**Sonya Huang**

什么时候应该构建专门的 RL environment，什么时候让实际应用本身充当 environment？

**Will Brown**

两种都有道理。如果你在训练一个 Rust coding 模型，你可能需要覆盖不同领域任务的多个 environment。如果产品本身就是面向终端用户的 agent 界面，让产品直接变成 environment 可能最合理。想做这种事情的公司通常也是那些关心自己该用 Claude 还是 GPT-5 的公司——他们已经在搭建评估 system prompt、model endpoint 的基础设施。而这恰好就是做 RL 需要的基础设施。

## Agent Harness 与 RL 的关系

**Sonya Huang**

Harness 是当前的热门话题。Claude Code 在 harness 上的工程做得特别好。Harness 和 RL 是什么关系？

**Will Brown**

Harness 是 environment 的一部分。对任何 eval 或 environment 任务，有输入，中间发生一堆事情，最后有输出状态需要评分。Harness 负责连接模型和系统中其他组件的交互方式。比如 coding agent 里，harness 就是 CLI agent，terminal 就是 environment。但这不是通用定义——有时候 harness 只是 system prompt 加工具，有时候会 spawn sub-agent。我们用 environment 作为抽象层，因为它能涵盖任何形式的系统与模型交互。

## Environment 的多种用途

**Sonya Huang**

所有公司都应该用 environment 做 post-training 吗？

**Will Brown**

Environment 是一种工具，可以做所有这些事。它不把你绑定到 RL 上。比如做 distillation：把大模型接到 environment 里跑很多次，用 grader 筛选最好的样本做 SFT。也可以做 prompt optimization，AB test 不同模型。核心理念是每家 AI 公司都应该在优化自己的 AI 系统。

**Sonya Huang**

换一种方式理解：eval 就是衡量你的 agent 在客户预期面对的那些 environment 中的表现。

**Will Brown**

对。用传统 ML 的话说，你不想 overfit 到 test set。但我们已经在接受用 eval 信号反馈到模型里。Environment 和 eval 基本是同一个东西。RL 就是这种迭代的规模化版本。

## RL 与 Post-training 的关系

**Sonya Huang**

你觉得 RL 和 post-training 是同义词吗？

**Will Brown**

RL 现在确实是主角——如果你在做大规模模型，RL 可能是花最多时间和算力的地方。但不是唯一要做的事。从初始模型到部署系统，还涉及 prompt tuning、SFT、online distillation 等很多算法。RL 是中间最大的那个，但周围还有很多工具。

## 谁能用好这些工具

**Sonya Huang**

是不是只有真正厉害的 AI researcher 才能用好？还是普通 Fortune 500 公司也能上手？

**Will Brown**

大部分公司都有能力。任何 Fortune 500 公司很可能都有 AI 工程师团队，他们跟得上最新工具，会用 Claude Code，对模型和 prompting 有见解。这就是我们的目标客户。

**Johannes Hagemann**

尤其是给他们合适的工具。可能那些大公司里没人能 debug TPU cluster，或者搞定大规模 agentic RL 需要的其他组件。所以我们把整个基础设施抽象掉，让他们用起来更简单。

## 客户案例：RCI 与医疗 AI

**Johannes Hagemann**

我们最喜欢的客户之一是 RCI，一个做前沿开源模型的新实验室。他们在用我们整个技术栈。我们一起训练了一些最大的 mixture of experts 模型并开源出来。在 post-training 方面也有深入合作。

**Will Brown**

RCI 是我们非常密切的合作伙伴。他们推动我们搭建了从 compute orchestration 到 inference 的各种组件。他们在 openness 理念上跟我们很一致，但更偏向直接面向企业交付。而我们的定位是在基础设施层提供更好的开发者体验。

在 Environment 产品方面，我们关注研究社区。很多研究生和刚入行的人在用，也有专注特定领域的实验室。比如医疗 AI——我们跟 OpenMed 等团队密切合作，他们想创建 benchmark 来评估模型的医疗能力，包括诊断、患者交互、医疗文献问答。他们在意赢得医疗专业人员的信任，想专注领域而不是 LM 基础设施。

## 游戏化 Environment 与 Residency 项目

**Sonya Huang**

有没有客户在做更偏 Atari 风格的从环境中学习的事情？

**Will Brown**

我们专注在 LLM 和 foundation model 上。但也有更像游戏的 environment。一个经典例子是 Wordle——它是一个很棒的 hello world environment。基础设施简单但表达力强，在几个 GPU 上跑一个小时就能看到模型学会把游戏玩得更好。

**Johannes Hagemann**

更偏玩具级的游戏例子通常是大家学习构建 RL environment 的起点，这也是 environment hub 大量使用的场景。另一个值得提的是我们的 RL residency——目前有 14 到 16 个成员，在各种垂直领域做出了很棒的 environment：可验证的软件工程、Lean 数学证明、医疗、物理、cybersecurity 等等。

## Environment 构建的成本与复杂度

**Sonya Huang**

我听说 foundation model 公司在这些 environment 上每个要花上百万美元。构建 cybersecurity environment 具体需要做什么？

**Will Brown**

Cybersecurity 领域有很多现成的 capture the flag 工具。一个系统里有隐藏 bug，本来是给程序员做 hackathon 的，但可以改造成给 LLM 用的。变成一个完整的 software environment，agent 在 terminal 里用 bash 命令和文件交互，完成后用代码评估状态。

我们发现大家对用 RL 评估数据质量很感兴趣。如果 environment 有后门，模型会利用它刷分。从创意到进入 frontier lab 的 training run，中间需要大量审查。做中等规模的 run 可以帮你发现问题在哪。

## Environment 中的 Sim-to-Real Gap

**Sonya Huang**

构建出来的 environment 能反映真实世界的复杂度吗？有没有 sim-to-real gap？

**Will Brown**

在复杂度层面没有障碍——原则上可以做到任何复杂度。真正的瓶颈是模拟器成本。关键是找到聪明的方法来 mock 系统中合适的部分。就像地图游戏里的迷雾——如果你知道哪些区域会被探索，就能决定哪些部分重要。比如 Tau Bench 是关于客服 agent 跟数据库交互的，你不需要百万条记录的完整数据库，一个只包含预期数据的 mock 数据库就够了。

## 平台设计哲学

**Sonya Huang**

你们的平台在高效创建 environment 方面有辅助吗？

**Will Brown**

这绝对是我们设计一切的方式。我们保持很多门开着——有基础层的通用 environment，然后可以往上扩展到 coding agent environment、带全局 sandbox 的、每个 rollout 一个的。我们也在想怎么提供好的开发者体验，比如文档和 skill file。做这类研究会积累大量 institutional knowledge，把这些知识嵌入产品、library 设计和文档中很重要。

## 从数据标注到 Environment 构建

**Sonya Huang**

你们读过 Rich Sutton 的 The Age of Experience 吗？你觉得构建 environment 是 Scale AI 时代数据标注的自然继承者吗？

**Will Brown**

确实看起来已经是了。主要实验室的重心在往这个方向转。人类数据不会消失，因为创建 environment 针对的就是模型还不够好的东西。关键是找到人类能最独特地帮助模型提升的信息点，建立从懂行的人到模型的知识流。目前 RL 是最有效的方式——通过任务、LM judge 的 prompt、success rubric。

## 为什么要做 Environment Hub

**Sonya Huang**

为什么要做一个 environment 的 hub？

**Johannes Hagemann**

最初是因为我们看到很多开源 repo 里塞满了各种 environment 实现。我们之前已经做了 Verifiers 框架来标准化流程。但超越单纯分享 environment，我们想要一个可以构建大量基础设施的地方——比如把 evaluation 跟 environment 深度集成，让大家能马上在所有 frontier model 上测试。我们有自己的大规模 trainer PRIME RL，但也开放地跟不同 trainer 集成。

## 社区行为与热门 Environment

**Sonya Huang**

社区行为模式是怎样的？有没有人 fork 和修改 environment？

**Will Brown**

很多人想保持 environment private，这可以理解。但 hub 的价值在于可以做对照实验、用统一实现的热门 benchmark 监控训练进度、检查是否能泛化到其他任务。最受欢迎的 environment 通常是文档示例里用到的。Wordle 很热门，但被 fork 和改造最多的是 Wiki Search——一个在 Wikipedia 上做搜索的模板，可以用于更广泛的 agentic search。你只需替换文档数据就行了。

## RL 的效率问题

**Sonya Huang**

Andrej Karpathy 说 RL 效率很低，像用吸管吸取 bit。你同意吗？研究上会有什么进展来让 RL 更高效？

**Will Brown**

确实 RL 用了大量算力获取相对较小的信号。但这也是它的价值所在。人类数据很难规模化，RL 让你用算力换数据，从少量数据中提取更多价值。而且在没有 golden example 的情况下，探索正是 RL 的核心。也许将来会有更高效的探索方式，但目前 RL 就是前沿。

**Johannes Hagemann**

我也很好奇 Andrej 的看法在最近 Claude Code 大火之后有没有变。我们不觉得 RL 是终点，一直想站在范式变革前沿。但我们也知道有局限性，比如 context 方面的硬限制。

## Recursive Language Model

**Sonya Huang**

DSPY 作者们有一个新东西——recursive language model。你们怎么看？

**Johannes Hagemann**

我们对更长 horizon 的 agent 很感兴趣，内部想了很长时间怎么让模型学会管理自己的 context。现在大家搭各种 scaffold 做 context management，但我们觉得更好的方向是让模型自己学会。Recursive language model 是最有前景的方向之一。

它给语言模型一个持久化 Python wrapper 里的变量，模型可以检索、转换数据、调用 sub-element 来管理自己的 context。已经证明能在超长 horizon reasoning benchmark 上大幅提升。我们接下来几个月要做的就是在这个 harness 里 training。

## Synthetic Data 与自主学习

**Sonya Huang**

研究领域还有什么让你们兴奋的？

**Will Brown**

Synthetic data 研究。感觉有很多事情应该能做到但还没出现——比如创造性的 self-reflection。大家都在讨论 continual learning，模型需要越来越擅长自己学东西。把 prompt optimization、distillation 和 synthetic data 用不同方式组合起来，让模型策划自己的 training data 甚至自己的 environment，搞清楚哪种方式对 lifelong learning 最有效。这个方向很值得深挖。

## 愿景：赋能每一家公司

**Sonya Huang**

如果一切顺利，世界会变成什么样？Prime Intellect 在其中扮演什么角色？

**Johannes Hagemann**

我们不希望 AI 在各个垂直领域创造的未来价值都只被大实验室拥有。我们要赋能创业者和企业，给他们工具来优化产品，不被碾压。创造更多的 Claude Code 时刻、Cursor for X 时刻。

**Will Brown**

某种程度上，是的——每家公司都会成为 AI lab。如果真正的专业知识是瓶颈，你有时候更想要那个干了 30 年的人。有些知识来自对问题的深度理解和长期互动，在短短的 prompt 里几乎不可能复制。你需要让 institutional knowledge 和 best practice 随着时间复利增长，站在之前的肩膀上，而不是每天从零开始。我们想让任何公司都能做到这一点。当编程门槛越来越低的时候，AI research 也会发生同样的事情。

**Sonya Huang**

这是一个非常鼓舞人心的愿景。非常感谢你们今天来做客，在 environment 和 environment hub 方面开创了先河。感谢你们揭开了 environment 的神秘面纱，分享了对未来的愿景。
