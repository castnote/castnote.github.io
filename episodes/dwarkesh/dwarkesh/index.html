Andrej Karpathy：AGI 仍需十年

本期节目中，Dwarkesh Patel 与前 Tesla AI 负责人、Eureka Labs 创始人 Andrej Karpathy 进行了一次深度对话。Karpathy 解释了为什么他认为通用人工智能至少还需要十年而非一年——当前的大语言模型虽然令人印象深刻，但在持续学习、多模态、计算机操作等方面仍有大量认知缺陷。两人回顾了 AI 领域 15 年来的重大转折（从 AlexNet 到强化学习再到 LLM），讨论了预训练与进化的类比、上下文学习的本质、强化学习的根本缺陷（「通过吸管吸取监督信号」）、模型坍缩与合成数据困境，以及认知核心应该有多大等前沿话题。Karpathy 还分享了他构建 NanoChat 的经验——为何编程模型在非模板化的创新代码上仍然力不从心，以及自动驾驶对 AI 部署时间线的启示。最后，Karpathy 谈到了他创办 Eureka Labs 的愿景——通过构建「知识坡道」来释放人类认知潜力，以及他作为教育者的教学方法论。

Dwarkesh Patel

今天和我对话的是 Andrej Karpathy。Andrej，你为什么说这将是「Agent 的十年」，而不是「Agent 之年」？

Andrej Karpathy

首先感谢你邀请我，很高兴能来。你提到的这句话——「Agent 的十年」——其实是对业界已有说法的一个回应。一些实验室暗示今年将是 Agent 之年，说的是大语言模型接下来的进化方向。我当时被这个说法刺激到了，因为我觉得行业里存在一些过度预测。在我看来，更准确的描述应该是「Agent 的十年」。我们确实已经有了一些非常早期但令人印象深刻的 Agent，我每天都在用——Claude、Codex 等等——但我觉得还有太多工作要做。所以我的反应是：我们会和这些东西共事一整个十年，它们会越来越好，这将是很棒的事情。但我只是在回应那个时间线的暗示。

Dwarkesh Patel

你觉得什么需要十年才能实现？瓶颈在哪里？

Andrej Karpathy

让它真正能用。当我们谈论 Agent 的时候——或者说实验室们心目中的 Agent，也许跟我想的差不多——你应该把它想象成一个员工，或者一个你雇来跟你一起工作的实习生。比如你这里有一些同事在帮你做事，你什么时候会愿意让 Claude 或 Codex 来代替他们做那些工作？目前它们当然做不到。需要什么才能让它们做到？你为什么今天不这么做？原因就是它们还不行。它们的智能不够，多模态能力不足，不能操作计算机，也缺少很多其他能力。它们没有持续学习的能力——你不能告诉它一件事然后指望它记住。它们在认知上有很多不足，就是还不够用。我觉得要解决所有这些问题大概需要十年。

Dwarkesh Patel

作为一个职业播客主持人和 AI 的旁观者，我能比较容易地指出缺什么——持续学习缺失、多模态缺失。但我没有什么好办法来判断时间线。如果有人问我持续学习需要多长时间？我没有任何先验经验来判断这是一个需要 5 年、10 年还是 50 年的项目。为什么是十年？为什么不是一年？为什么不是五十年？

Andrej Karpathy

这确实涉及到一些我个人的直觉，同时也是基于我自身经验的外推。我在 AI 领域已经待了将近二十年——大概有 15 年左右吧。你之前采访过 Richard Sutton，他在这个领域的时间当然更长。但我确实有大约 15 年的经验，看过各种预测，也看过它们实际上是怎么兑现的。我既做过研究也在工业界工作过，所以我有一种从这些经历中积累的总体直觉。我觉得这些问题是可以解决的、是可行的，但仍然很难。如果我做个平均，十年就是我的感觉。

Dwarkesh Patel

这其实非常有趣。我想听你讲的不仅是历史，还有在那些突破时刻，身处其中的人觉得接下来会发生什么？他们的判断在哪些方面过于悲观，又在哪些方面过于乐观？

AI 发展史回顾

Andrej Karpathy

要一个个讲吗？这是个巨大的问题，毕竟涉及 15 年发生的事情。AI 真的非常精彩，因为出现了好几次我称之为「地震级」的转变——整个领域突然看起来完全不同了。我大概经历了两三次这样的转变，而且我觉得未来还会有更多，因为它们总是以一种几乎令人惊讶的不规则性出现。

我的职业生涯开始时，纯粹是因为机缘巧合——我在多伦多大学紧挨着 Jeff Hinton。Jeff Hinton 当然是 AI 领域的教父级人物，他在训练各种神经网络，我觉得这太酷了。但当时这绝对不是 AI 领域的主流方向，这是一个边缘小众领域。

第一次剧烈转变是 AlexNet 带来的。AlexNet 基本上重新定位了整个领域，所有人都开始训练神经网络。但当时还是非常「按任务」来的——也许我有一个图像分类器，或者一个神经机器翻译模型之类的。后来人们逐渐开始对 Agent 产生兴趣。人们开始想：好的，也许我们已经给视觉皮层打了勾，但大脑的其他部分呢？怎么得到一个完整的 Agent 或完整的实体，能够在世界中实际交互？

2013 年前后的 Atari 深度强化学习就是这种早期 Agent 努力的一部分——试图让 Agent 不仅能感知世界，还能采取行动、与环境交互并获得奖励。当时的目标是 Atari 游戏。我觉得那其实是一个弯路。早期的 OpenAI 也采纳了这个方向——大家都在做强化学习、玩游戏、打败各种游戏。

我一直对「用游戏通向 AGI」这件事有些怀疑，因为在我看来，你真正想要的是类似会计师那样的东西——能与真实世界交互的。我看不出游戏怎么能加起来变成那个。所以我在 OpenAI 的项目——在 Universe 项目的框架下——是一个用键盘和鼠标操作网页的 Agent。我真的想做一个能与数字世界交互、能做知识工作的东西。但事实证明，这太早了，早到我们根本不应该做那个。因为如果你只是在那里跌跌撞撞地胡乱点击键盘和鼠标、试图在这些环境里获得奖励，奖励太稀疏了，你什么也学不到，会烧掉一整片森林的算力，却永远做不出任何东西。

你缺少的是神经网络中的表示能力。今天人们在训练那些计算机操作 Agent，但他们是在大语言模型的基础上做的。你实际上必须先有语言模型，先有表示能力，先做完所有预训练和 LLM 的工作。所以我觉得大致来说，人们反复过早地尝试做完整的东西——人们真的试图过早地追逐 Agent，这就是 Atari、Universe，以及我自己的经历。你实际上必须先做一些其他事情，然后才能做 Agent。现在 Agent 的能力强多了，但也许我们还缺少这个技术栈的某些部分。

我觉得这大概就是三大类：按任务训练神经网络；第一轮 Agent 尝试；然后是 LLM——先追求神经网络的表示能力，再在上面叠加其他东西。

构建幽灵而非动物

Dwarkesh Patel

如果我要为「突然范式」做辩护的话：人类确实可以同时处理一切，动物也能同时处理一切。动物可能是更好的例子，因为它们甚至没有语言的脚手架——它们就是被扔到世界里，必须在没有任何标签的情况下理解一切。那么 AGI 的愿景是不是就应该是一个东西，直接看着感官数据、看着屏幕，然后从零开始搞明白发生了什么？如果一个人被放在类似的情况下，需要从零开始训练——这就像人类成长或动物成长一样。为什么这不应该是 AI 的愿景，而要像我们现在这样做数百万年的训练呢？

Andrej Karpathy

我觉得这是个非常好的问题。我在与动物做类比时非常谨慎，因为动物是通过一个完全不同的优化过程产生的。动物是进化出来的，它们实际上自带大量内建的硬件。我在博客文章里用的例子是斑马——斑马出生后几分钟就能跑起来跟着妈妈走。这是一件极其复杂的事情。那不是强化学习，那是内建的东西。进化显然有某种方式把我们神经网络的权重编码在 ATCG 里面，我完全不知道那是怎么做到的，但显然是可以的。

所以我觉得，大脑来自一个完全不同的过程。我非常犹豫从中汲取灵感，因为我们实际上没有在运行那个过程。所以在我的文章里，我说我们不是在构建动物，我们是在构建幽灵。或者叫灵体之类的。因为我们不是通过进化来训练的，我们是通过模仿人类以及他们放在互联网上的数据来训练的。所以最终你得到的是这些飘渺的灵体——它们完全是数字化的，在模仿人类。这是一种不同类型的智能。如果你想象一个智能空间，我们是从一个不同的起点出发的。我们不是在构建动物。但我觉得随着时间推移，让它们变得更像动物也是可能的，而且我们应该这么做。

Sutton 的框架基本上就是「我们要构建动物」。如果我们能让那个思路成功，那将是了不起的。如果有一种算法你可以直接在互联网上运行，让它学会一切，那将是不可思议的。但我几乎怀疑那个东西并不存在。而且动物确实也不是这么做的，因为动物有进化这个外循环。很多看起来像「学习」的东西其实更多是大脑的成熟，我觉得动物真正做的强化学习非常少，而且主要是运动任务——不是智能任务。所以我觉得人类大致上并不怎么使用强化学习。

Dwarkesh Patel

让我消化一下，这里面有很多不同的想法。你是不是在说进化做的事情类似于预训练——构建一个能够理解世界的东西？区别在于进化必须通过 3GB 的 DNA 来传递。这和模型的权重完全不同——大脑的权重显然不是编码在精子和卵子里的。所以它必须生长出来，而且大脑里每个突触的信息根本不可能全部存在于 DNA 的 3GB 数据里。进化似乎更接近于找到那个「算法」，然后这个算法再做终身学习。也许终身学习不等同于强化学习。这和你说的一致吗？

Andrej Karpathy

我想是的。一定有某种神奇的压缩在发生，因为神经网络的权重显然不是存储在 ATCG 里面的。有某种极度压缩，还有某种学习算法被编码在里面，能够接管并在线完成部分学习。我同意你这一点。

但基本上我的出发点更加实用主义。我不是从「让我们构建动物」的角度出发，而是「让我们构建有用的东西」。我戴着安全帽在观察：我们不会去做进化，因为我不知道怎么做。但我们确实可以通过模仿互联网文档来构建这些幽灵般的灵体。这行得通，而且实际上它是一种积累大量内建知识和智能的方式——某种程度上类似于进化所做的。所以我把预训练叫做一种「劣化版进化」——在我们现有的技术和资源下，这是达到一个起点的实际可行方式，然后我们才能在上面做强化学习之类的事情。

Dwarkesh Patel

让我为另一方做辩护。进化并不是给我们知识，它给我们的是找到知识的「算法」。这和预训练不一样。如果说进化和预训练都是给我们知识，那类比就不成立了。但如果说预训练帮助构建了一种能更好学习的实体——它教会了元学习——那也许是类似于找到一种算法。

Andrej Karpathy

你说得对。预训练实际上做了两件相当不同的事情。第一，它积累了所有这些知识。第二，它通过观察互联网上的算法模式，实际上变得智能了——它在神经网络内部启动了各种小电路和算法来做上下文学习（In-Context Learning）之类的事。

实际上，你并不需要也不想要那些知识。我甚至觉得知识可能在拖后腿，因为它让神经网络有时过于依赖知识。比如 Agent 不太擅长处理超出互联网数据分布的东西。如果它们知识少一点、记忆少一点，也许反而会更好。

所以我觉得接下来的研究方向之一是：我们需要找到方法去除部分知识，保留我所说的「认知核心」——一个被剥离了知识但保留了算法、保留了智能的魔力、保留了问题解决策略的聪明实体。

上下文学习与梯度下降

Dwarkesh Patel

这里面有太多有趣的东西了。先从上下文学习说起。这是一个很明显但值得拿出来细想的观点：这些模型表现得最聪明的时候——当我和它们对话时会觉得「哇，另一端确实有什么东西在回应我、在思考」——都是发生在上下文中。那就是你能直观看到真正智能的地方。而上下文学习过程是通过预训练的梯度下降自发涌现的。它自发地元学习了上下文学习，但上下文学习本身不是梯度下降——就像人类的终身智能受进化制约，但我们一生中的实际学习是通过某种其他过程进行的。

Andrej Karpathy

我其实不完全同意这一点。我不太愿意说上下文学习没有在做梯度下降。它没有做显式的梯度下降，但上下文学习基本上是在 token 窗口内进行模式补全。互联网上有海量的模式，模型学会了补全这些模式。模式匹配是在权重里完成的——权重在试图发现模式并完成模式，神经网络内部发生着某种神奇的适应。

有一些论文很有意思，它们实际考察了上下文学习的机制。有一篇论文用上下文学习做线性回归——你的输入是 XY 对，恰好在一条线上，然后你给一个 X，期望得到 Y。用这种方式训练出来的神经网络确实在做线性回归。而在普通线性回归中，你有一个小的梯度下降优化器来计算梯度并更新权重。当他们检查那个上下文学习算法的权重时，确实发现了与梯度下降机制的类似性。那篇论文的结论甚至更强——他们实际上手动设定了神经网络的权重，通过注意力机制（Attention）和网络内部结构来执行梯度下降。

所以我只是在推回你说的「上下文学习不是梯度下降」。谁知道上下文学习到底在做什么呢，但它可能在做某种变形的梯度下降。

Dwarkesh Patel

那值得想一想：如果上下文学习和预训练都在做某种梯度下降，为什么上下文学习感觉更像是在接近那种持续学习、真正智能的东西，而预训练给你的感觉没有那么强？如果是同一种算法，区别可能在哪里？

一种思路是看模型每接收一个信息能存储多少。预训练方面，以 Llama 3 为例，它在 15 万亿个 token 上训练，700 亿参数的模型相当于每个 token 只存储了 0.7 比特。而在上下文学习中，KV 缓存每个额外 token 增长大约 320KB——这是一个 3500 万倍的差异。

Andrej Karpathy

我大致同意。我通常的说法是：训练时发生的一切，在网络中只是一种「模糊的回忆」，因为压缩极其剧烈——你把 15 万亿个 token 压缩到了只有几十亿参数的最终网络里。所以这是一种对互联网文档的模糊回忆。而在上下文窗口中发生的一切——你把所有 token 接入，它在构建 KV 缓存表示——对神经网络来说是非常直接可达的。所以我把 KV 缓存和测试时发生的东西类比为工作记忆。

LLM 和人类之间总是有这些几乎令人惊讶的类比。权重里的东西就像你一年前读过的东西的模糊回忆，你在上下文中给它的东西就直接在工作记忆里了。所以你去问 LLM 某本书里发生了什么，它会给你一些大致正确的东西。但如果你把整章放进上下文窗口再问问题，结果会好得多，因为现在它加载到了模型的工作记忆里。

大脑类比与认知缺陷

Dwarkesh Patel

退一步来看，人类智能中哪些方面是我们最未能复制的？

Andrej Karpathy

还有很多方面没做到。做一些不完美的类比：Transformer 神经网络极其强大、极其通用——你可以在音频、视频、文本上训练 Transformer，它就是学习模式，效果非常好。这让我觉得它有点像大脑皮层组织——因为大脑皮层出了名的可塑性强，你可以重新连接大脑的部分区域，比如把视觉皮层重连到听觉皮层，动物照样能适应。

当我们在神经网络里做推理和规划——思维链模型——那有点像前额叶皮层。也许这些可以打个勾。但还有很多大脑部位和核团没被探索。也许基底神经节在做一点强化学习——对应模型的 RLHF 微调。但海马体对应什么不明显。有些部分可能不重要——也许小脑对认知不重要。但杏仁核、所有的情感和本能——还有很多古老的核团——我觉得我们没有真正复制。

另一种回答是：你不会雇这个东西当实习生。它有太多认知缺陷，我们在和模型对话时都能直觉感受到。它就是还没到那个程度。

Dwarkesh Patel

有人对持续学习说：就像上下文学习从预训练中自发涌现一样，如果模型被激励在更长的时间跨度上回忆信息——比如有一个外循环的强化学习包含很多会话——那持续学习也会自发涌现。模型会自己学会微调自身或写入外部记忆之类的。你觉得这可行吗？

Andrej Karpathy

我不太认同这个说法，因为这些模型每次启动、上下文窗口里 0 个 token 的时候，总是从头开始回到它们原来的状态。我不确定在那个世界观里这会是什么样子。

做个类比：我觉得醒着的时候我在积累一个上下文窗口，白天发生的事都在里面。但我觉得睡觉时有什么神奇的事情发生了——那个上下文窗口并没有保留。我觉得有某种蒸馏过程，把信息蒸馏到大脑的权重里。这发生在睡眠期间。我们在大语言模型里没有这个等价物。这在我看来就是持续学习缺失的核心——模型没有这个「蒸馏阶段」来消化发生过的事、反复思考、做某种合成数据生成并蒸馏回权重。

也许我们需要为每个人定制神经网络——可能是一个 LoRA，不是完整的权重变化，只是一小部分稀疏的权重子集被改变。但基本上我们确实想创造这些拥有非常长上下文的个体。不能只靠留在上下文窗口里——虽然窗口在变得很长，也许我们会有某种精巧的稀疏注意力机制——但人类显然有某种把部分知识蒸馏到权重里的过程，我们缺少这个。

人类大概也有某种非常精巧的稀疏注意力方案。DeepSeek V3.2 刚出来，我看到他们有稀疏注意力，这是拥有超长上下文窗口的一种方式。所以我觉得我们在通过一个完全不同的过程重新发明进化想出的很多认知技巧，但我们可能会在认知架构上趋同。

Dwarkesh Patel

10 年后你觉得还会是某种 Transformer 吗？也许有更改进的注意力和更稀疏的 MLP？

Andrej Karpathy

我喜欢做时间平移不变性的思考。10 年前——2015 年——我们主要用卷积神经网络，残差网络刚出来。和现在惊人地相似，但也很不同——Transformer 还没出现。所以也许可以打赌的是：10 年后我们仍然用梯度下降训练巨大的神经网络，只是看起来有点不同，一切都大得多。

我前段时间做过一个有趣的练习——回到 1989 年，重现 Yann LeCun 1989 年的卷积网络，那是我所知的第一个用梯度下降训练的现代神经网络，做数字识别的。我感兴趣的是：进步中有多少来自算法、多少来自数据、多少来自算力和系统？只靠「时间旅行 33 年」带回去的算法知识调整 LeCun 当年的设置，我能把错误率减半。但要进一步提升就需要大量更多数据——训练集增加 10 倍——然后还需要更多计算优化、更长的训练时间加上 Dropout 等正则化技术。

所有这些东西必须同时进步。我们可能会有更多数据、更好的硬件、更好的内核和软件、更好的算法——它们几乎出奇地均等。这个趋势持续了很久。所以我的猜测是：10 年后可能还是用梯度下降训练的巨型神经网络。

NanoChat 与编程

Dwarkesh Patel

你最近刚写完 NanoChat 的代码，构建聊天机器人的每一步对你来说都是新鲜的。你在组装这个仓库的过程中有什么令人惊讶的收获吗？

Andrej Karpathy

NanoChat 是我前两天刚发布的一个仓库。它试图成为最简单、最完整的、端到端覆盖构建 ChatGPT 克隆全流程的仓库。就学习收获而言，与其说我发现了新东西，不如说我已经知道该怎么做了，这只是一个机械地把它构建出来、让它足够干净以便人们可以学习的过程。

Dwarkesh Patel

学习这个仓库最好的方式是什么？删掉所有代码自己从头重写？

Andrej Karpathy

这是个好问题。基本上它大约 8000 行代码，覆盖了整个流程。我的建议是：把它放在右边的显示器上，你在左边的显示器上从头构建。你可以参考它，但不能复制粘贴。

不过这个仓库本身也是个相当大的东西。你写这种代码不是从上到下写的——你从一些代码块开始，然后让这些块逐渐增长。这个信息是缺失的——你不会知道从哪里开始。所以不仅需要最终的仓库，还需要构建仓库的过程，这是一个复杂的「块增长」过程。这部分还没有——我想稍后可能以视频的形式补上。

但大致来说，自己构建东西，不允许复制粘贴。我确实认为知识有两种——有高层的表面知识，但当你从头构建东西时，你被迫面对你实际上不理解的地方，而且你不知道自己不理解。这总是会带来更深的理解。如果我不能构建它，我就不理解它——我一直非常坚信这一点。所以：不要写博客文章，不要做幻灯片，不要做任何那些。构建代码，组织它，让它跑起来。这是唯一的路。否则你的知识是有缺口的。

Dwarkesh Patel

你在推特上说编程模型对你组装这个仓库帮助不大？

Andrej Karpathy

这个仓库我用了一个多月来构建。我觉得现在人们与代码交互有三大类方式：完全拒绝 LLM 自己从头写——我觉得这已经不对了；中间层——也就是我的位置——你还是自己写很多东西，但使用自动补全，你开始打前几个字母，模型帮你补全，大多数时候是对的，你还是代码的架构师；然后是 vibe coding——「嘿，请帮我实现这个」，回车，让模型来做，这就是 Agent。

Agent 在某些场景下很好用。比如做模板化代码——大量复制粘贴的样板代码，互联网上有很多同类代码的例子，模型对这些非常擅长。但 NanoChat 不是这种情况——它是一个相当独特的仓库，没有太多类似的代码，不是模板代码，而是智力密集型代码，一切都需要非常精确地安排。

模型有太多认知缺陷。一个例子是：它们一直以为我在写普通代码，但我不是。比如同步 8 个 GPU 之间的梯度，通常用 PyTorch 的 DDP 容器。但我没有用 DDP，因为我不需要它——我自己写了同步逻辑放在优化器的 step 里。模型一直试图让我用 DDP 容器，它们就是无法内化「你有自己的自定义实现」这个事实。

然后它们不断试图把风格搞砸——它们过度防御性地写代码，到处加 try-catch，不断试图做成生产级代码库。我的代码里有一堆假设条件，这没问题，我不需要那些额外的东西。它们在膨胀代码库、膨胀复杂度、持续误解、使用过时的 API。很乱。我可以进去清理，但帮助不大。

而且要用英语打字描述我想要什么——太多打字了。如果我直接导航到代码中我想要的位置，开始输入前几个字母，自动补全就能给你代码。这是一种非常高信息带宽的方式来指定你想要什么——你指向代码中需要插入的位置，打出前几个字符，模型就会补全。

不过也有两个地方我确实用了模型。一个是生成报告——那更像模板工作，vibe coding 一部分，效果还行。另一个是用 Rust 重写 tokenizer——我对 Rust 不太熟，所以在写 Rust 代码时有些 vibe coding 的成分。但我有完全理解的 Python 实现，只是在做更高效的版本，而且有测试，所以感觉比较安全。模型降低了你进入不太熟悉的语言或范式的门槛，这方面确实很有帮助。

AI 对软件工程的影响

Dwarkesh Patel

这个问题之所以有趣，是因为人们关于 AI 爆发式发展达到超级智能的主要叙事就是：AI 自动化了 AI 工程和 AI 研究。人们会看到 Claude Code 能从零搭建整个应用，然后想：如果在 OpenAI 和 DeepMind 内部有这种能力，想象一下一百万个你在并行工作，找到各种小的架构改进——所以听到你说这恰恰是它们不对称地弱的领域，对于预测 AI 2027 式的智能爆炸是否会很快发生非常有参考价值。

Andrej Karpathy

我觉得你说得很对，这也是为什么我的时间线长一些。它们不擅长写「从未被写过的代码」——也许可以这么概括，而这恰恰是我们构建这些模型时要做的事。

Dwarkesh Patel

但是你加到 NanoChat 里的那些架构改进，都在论文里甚至在某些代码库里存在。模型应该能整合这些才对，对吧？

Andrej Karpathy

很难说。它们有一些知识但不完全懂，不知道怎么把它完整地集成到你的仓库里、你的风格里、你的代码里、你做的那些自定义设定和所有假设条件里。它们有一些知识但还没到能实际整合、理解并正确应用的地步。

这些东西一直在进步。现在最先进的模型大概是 GPT-5 Pro。如果我有 20 分钟，我会把整个仓库复制粘贴给 GPT-5 Pro 这个「神谕」问一些问题，结果通常不错，比一年前好得多。但总体来说模型还不够好。我觉得行业有些过度跳跃，试图假装这些东西很厉害——其实不然。很多是粗糙的产物，行业没有正视这一点，也许是在为融资造势。我们处在一个中间阶段：模型很惊艳，但仍需要大量工作。目前自动补全是我的最佳平衡点，但某些类型的代码我会用 Agent。

Dwarkesh Patel

在编程史上有过很多生产力提升——编译器、linting、更好的编程语言——它们提高了程序员的生产力但没有导致爆发。这听起来很像自动补全 Tab。另一个类别是程序员本身的自动化。听起来你看到的更多是历史上「更好的编译器」那种类比？

Andrej Karpathy

我确实很难区分 AI 在哪里开始和结束，因为我认为 AI 从根本上是计算的延伸。我看到一个连续谱——从代码编辑器、语法高亮、类型检查，到搜索引擎，甚至 Google 早期就自认为是一家 AI 公司。这些递归自我改进一直在发生。现在我们有了更好的自动补全，也有了一些 Agent——它们是这种循环的东西但有时会跑偏。正在发生的是人类渐渐地做越来越少的底层工作——就像我们不写汇编了因为有编译器一样。我们在自动化之上非常缓慢地抽象自己。我称之为「自主性滑块」——越来越多能被自动化的东西被自动化了。

强化学习的根本问题

Dwarkesh Patel

谈谈强化学习。人类是怎么仅仅通过与环境交互就建立起丰富的世界模型的？比如一个人创业，10 年后才知道成功还是失败，我们说她获得了大量智慧和经验。但这不是因为过去 10 年每件事的对数概率都被更新了——发生的是更深思熟虑、更丰富的东西。这在机器学习中的类比是什么？

Andrej Karpathy

也许我的说法是：人类不用强化学习。人类做的是不同的东西。强化学习比普通人以为的要差得多。强化学习很糟糕，只是因为我们之前有的东西更差。

在强化学习中，比如你在解一道数学题。你先并行尝试几百种不同的方法——可以很复杂，试这个试那个，这个不行那个不行。然后也许你得到一个答案，你查看参考答案，发现这些对了但那 97 个不对。

强化学习做的事情就是：找到对的那些，然后一路上的每一个 token 都被上调——「多做这样的事」。问题是——这基本假设你在走对路的过程中每一步都是对的，但事实并非如此。你可能走了很多错误的弯路才最终到达正确答案，所有那些错误的步骤只要最终答案对了，都会被当作好的做法来强化。

这太糟了。你做了所有那些工作，最后只得到一个数字——「你做对了」。然后基于这一个数字，你给整条轨迹打上正面或负面标签。你在通过一根吸管吸取监督信号——你做了可能是好几分钟的推演，然后通过这根吸管把最终奖励信号的几个比特吸出来，广播到整条轨迹上，用它来上调或下调那条轨迹。

这太疯狂了。人类绝不会这么做。第一，人类不会做几百次并行尝试。第二，当一个人找到答案后，他们会有一个相当复杂的回顾过程——「我觉得这些部分做得好，那些部分做得不好，我应该怎样怎样」。他们会思考。目前的 LLM 里没有任何东西做这件事。但我看到有论文开始尝试了，因为这对领域里的每个人来说都是显而易见的。

回顾一下进展：首先是模仿学习——在预训练模型上用人类对话数据微调，变成助手。InstructGPT 那篇论文让我大开眼界，因为它指出你可以取预训练模型（本质上是自动补全），只要在看起来像对话的文本上微调几轮，模型就能非常快地调整成对话模式，同时保留预训练的所有知识。然后来了强化学习，它允许你做得比纯模仿更好，因为你可以有奖励函数并在上面爬坡。模型还能发现人类可能永远想不到的解决方案。这很了不起。但它仍然很蠢。我们需要更多东西。

Dwarkesh Patel

你把结果奖励的问题讲得很清楚了——你有一条巨长的轨迹，最后你试图从那一个最终比特中学到所有东西。为什么作为替代方案的「过程监督」（Process-Based Supervision）没有成功地让模型变得更强？是什么阻碍了我们使用这个范式？

Andrej Karpathy

过程监督就是说我们不只在最后才给你一个奖励，而是在每一步都告诉你做得怎么样。我们之所以没有这个，是因为怎么正确地做这件事并不简单。你有的是部分解答，你不知道怎么分配功劳。当你得到正确答案时，就是一个等式匹配——很简单。但如果你在做过程监督，怎么以可自动化的方式给部分解答分配功劳？这不明显。

很多实验室在试图用 LLM 做评判者——提示一个 LLM：「看看学生的部分解答，你觉得他们做得怎么样？」然后调整提示。但这里有一个相当微妙的问题：每当你用 LLM 来分配奖励时，这些 LLM 是有几十亿参数的巨大模型，它们是可被博弈的。如果你对它们做强化学习，几乎可以保证你会找到针对这些 LLM 评判者的对抗样本。做 10 步 20 步也许可以，但不能做一百或一千步，因为模型会找到各种缝隙和裂缝来欺骗评判者。

我印象中有一个突出的例子：我们用 LLM 评判者来做奖励——给它一个学生的解答，问它学生做对了没有。我们对着那个奖励函数做强化学习，效果很好。然后突然奖励变得极高——完美分——你一看觉得「哇，学生在所有问题上都是满分，数学完全解决了」。但实际上，当你看那些模型的输出，它们完全是胡说八道——开头还行，然后变成「嘟嘟嘟嘟」一堆乱码。

你看那个 LLM 评判者，发现那些乱码是评判模型的对抗样本，它给出 100% 的概率。因为这是训练分布之外的样本，模型从没见过，在纯泛化领域里你可以找到这种样本把它击破。你基本上是在训练一个「提示注入模型」——甚至不是提示注入，提示注入太高级了——你找到的是对抗样本，这些明显错误的解答但模型认为完美。

Dwarkesh Patel

如果你认为这是让强化学习更有效的瓶颈，那就需要让 LLM 成为更好的评判者。这是不是某种类似 GAN 的方法——你得训练模型变得更鲁棒？

Andrej Karpathy

最明显的做法是：那个乱码不应该得到 100% 的奖励——好，把它放进 LLM 评判者的训练集里说这是 0% 而不是 100%。你可以这么做，但每次你这么做就得到一个新的 LLM，它仍然有对抗样本。对抗样本有无穷多个。如果你迭代几次，可能会越来越难找到真正的对抗样本，但我不 100% 确定，因为这东西有万亿个参数。我猜实验室们都在尝试，但我仍然觉得我们需要其他想法。

Dwarkesh Patel

你对那种「其他想法」有什么大致的轮廓吗？

Andrej Karpathy

大致是这种「回顾解答然后生成合成样本、训练后变好、以某种方式元学习」的思路。有一些论文开始出现，但我只是在读摘要阶段，因为很多论文只是想法，还需要有人真正在前沿 LLM 实验室的规模上让它跑通。这些论文是有趣的想法，但我还没看到有人令人信服地证明它可行。不过实验室们相当封闭，谁知道他们现在在做什么。

Dwarkesh Patel

你暗示了人类做的另一件事可能不是「给自己出假题」，而是「反思」。睡觉或白日梦可能就是这个——不是自己出题，而是单纯地回顾。我不确定机器学习里「白日梦」或「睡眠中的反思」对应什么。

Andrej Karpathy

我确实觉得我们在这方面有欠缺。比如你在读一本书——目前 LLM 读书的方式是把文本展开，模型预测下一个 token，从中获取一些知识。但人类不是这么做的。我几乎觉得书不是我应该去关注和训练的叙述——书是给我提供的一组「提示」，让我去做合成数据生成。或者你去读书会跟朋友讨论。通过操作那些信息你才真正获得知识。我们在 LLM 里没有这个等价物。

我很想在预训练中看到某种阶段，模型会去「思考」材料、试图与它已知的东西调和、花一些时间去琢磨。我们目前什么都没有。这全是研究问题。而且有一些非常微妙的原因导致它不平凡。

比如，为什么我们不能就合成生成然后训练？每个合成样本你看上去会觉得「这看起来不错，为什么不能用来训练？」你可以试，但如果持续这么做模型会变得更差。这是因为所有从模型中采样得到的样本都是「静默坍缩」的——任何单个样本看起来都不错，但它们占据的是可能思想空间中极其微小的流形。

模型坍缩

Andrej Karpathy

一个简单的检验方式：去 ChatGPT 说「给我讲个笑话」，它只有大概三个笑话。它不会给你所有可能笑话的丰富多样性，它只知道三个笑话。它们是静默坍缩的。你得不到从这些模型中得到的丰富性、多样性和熵——跟人类相比。人类更嘈杂一些，但至少没有偏差——在统计意义上没有静默坍缩，保持了大量的熵。

Dwarkesh Patel

坍缩与合成数据生成的关系是——你想要能产生不在你数据分布中的合成问题或反思？

Andrej Karpathy

比如我们有一章书，我让模型去思考它，它会给你一个看起来很合理的东西。但如果我问 10 次，你会发现它们全一样。你不能仅仅靠在同样的提示信息上「缩放反思」就得到回报。任何单个样本看起来可以，但分布非常糟糕，糟糕到如果你持续在自己的产出上训练，模型实际上会坍缩。

我觉得这个问题可能没有根本性的解决方案。我也觉得人类随着时间也会坍缩。这些类比出奇地好——孩子们还没有过拟合，他们会说出让你震惊的话，因为他们还没有坍缩。但我们是坍缩的——我们最终反复思考同样的想法，说越来越同质的东西，学习率降低，坍缩持续恶化，然后一切走下坡路。

Dwarkesh Patel

有一篇非常有趣的论文说做梦是防止这种过拟合和坍缩的方式——做梦之所以有进化适应性，是因为它把你放在与日常现实非常不同的奇怪情境中，从而防止过拟合。

Andrej Karpathy

这是个有趣的想法。我确实觉得当你在脑中生成东西然后关注它时，你有点像是在训练自己的合成数据。如果做太久就会跑偏，坍缩太严重。所以你总是要在生活中寻找熵——跟其他人交谈就是一个很好的熵来源。也许大脑内部也建立了一些增加熵的机制。

Dwarkesh Patel

这是一个不太成形的想法。我们知道的最好的学习者是儿童，但他们极其不擅长回忆信息。事实上在最早期阶段你会忘记一切。但你极其擅长学习新语言和从世界中学习。也许这里面有某种「见林不见树」的元素——如果你什么都记住了反而看不到更大的模式。而 LLM 的预训练处于另一个极端——它们能逐字背出维基百科下一个词是什么，但快速学习抽象概念的能力远不如孩子。成人介于两者之间。

Andrej Karpathy

我觉得这里面确实有非常有趣的东西。人类确实更擅长「见林不见树」，而且我们其实不太擅长记忆——这实际上是一个特性而非缺陷。因为不擅长记忆，我们实际上被迫以更一般化的方式去发现模式。

LLM 的记忆力极强——你给它一段完全随机的序列，哪怕只训练一两次迭代它就能完整复述。没有人能读一次随机数序列就背出来。这对我们来说是特性而非缺陷，因为它迫使你只学习可泛化的成分。而 LLM 被它们对预训练文档的所有记忆分散了注意力。

这就是为什么当我谈到认知核心时，我实际上想去除记忆。我希望它们记忆更少，需要查东西就去查，只保留思考的算法、实验的概念、所有认知胶水。

认知核心的大小

Dwarkesh Patel

最优的智能核心最终需要多少比特？

Andrej Karpathy

历史上这个领域有一段时间非常信奉「规模即一切」——要做万亿参数模型。但实际上模型大小先是涨上去然后又下来了，而且即便现在它们也记忆了太多东西。我之前有个预测：我觉得我们可以得到十亿参数量级的非常好的认知核心。在 20 年后，如果你和一个十亿参数的模型对话，我觉得你能有一次非常有成效的对话。它能思考，更像人类。但如果你问它事实性问题，它可能得去查——但它知道自己不知道，会去查，会做所有合理的事情。

Dwarkesh Patel

十亿参数让我有些吃惊，因为我们已经有了十亿参数级别的模型而且相当聪明了。考虑到 GPT o-series 那种 200 亿参数的模型就比原始的万亿参数以上的 GPT-4 强得多，10 年后认知核心不应该更小吗？几千万甚至几百万参数？

Andrej Karpathy

你比我还激进。问题是训练数据——互联网——质量太差了。当你和我想到互联网时，你想的是华尔街日报那样的内容。但实际看前沿实验室的预训练数据集，随机抽一篇文档看，那真的是垃圾——什么股票代码、大量各种角落里的烂东西。不是华尔街日报文章——那种极其稀少。因为互联网太差了，我们实际上必须建非常大的模型来压缩所有这些。大部分压缩是记忆工作而不是认知工作。

我们真正想要的是认知部分——删掉记忆。所以我的意思是：我们需要聪明的模型来帮助我们精炼预训练数据集，把它缩减到只剩认知成分，然后你就能用更小的模型搞定。蒸馏效果极好，几乎每个小模型都是蒸馏来的。也许十亿参数还是有必要的，因为至少需要一些基本课程知识在里面——你不想让它什么都查。但它不需要有那种偏门冷知识。

Dwarkesh Patel

前沿模型的规模呢——训练出来的最大模型未来会更大、更小还是差不多？

Andrej Karpathy

我不确定我有强烈的预测。实验室们只是在务实地分配算力预算和成本预算。事实证明预训练不是你最应该投入算力的地方——所以模型变小了，因为预训练阶段变小了，但通过强化学习和各种后续训练来弥补。它们只是在各阶段之间实用地分配资源以获得最大回报。

我的基本预期是还有大量低垂的果实——数据集会好得多（当你看平均数据集它们实在太差了），硬件会更好，所有内核更好地利用芯片，所有算法在优化、架构和建模组件上都会改进。没有任何单一因素占主导——一切都进步 20%——这就是我一直看到的趋势。

AI 对经济的影响

Dwarkesh Patel

如果我们相信当前范式正在通向 AGI，这个进程的方式非常不像你天真预期的——你以为会是一点点拿走顾问的任务、一点点拿走会计的任务。但实际上，如果你看这些公司的收入（刨去普通聊天收入），API 收入几乎全被编程占据了。这个号称「通用」的东西压倒性地只在做编程。这是一个出人意料的 AGI 部署方式。

Andrej Karpathy

编程确实是 LLM 和 Agent 的完美首个应用场景——因为编程从一开始就围绕文本运作，是终端和文本，一切都基于文本。LLM 在互联网上训练，热爱文本，它们是完美的文本处理器。而且我们有大量为处理代码和文本预建的基础设施——比如 VS Code 可以显示代码，Agent 可以插入其中，代码变更可以用 diff 展示。我们为代码预建了大量基础设施。

相比之下，试图自动化幻灯片就难得多——因为幻灯片不是文本，是小图形、有空间排布、有视觉元素。如果 Agent 修改了你的幻灯片，怎么展示差异？没有工具展示幻灯片的 diff。所以有些东西对目前作为文本处理器的 AI 并不友好，而代码出人意料地友好。

Dwarkesh Patel

我不确定仅凭「代码是文本」能解释一切。我个人在纯语言输入输出的领域也试过让 LLM 派上用场——比如改写采访文字稿、基于文字稿剪辑片段——效果并不好。我们的共同朋友 Andy Matuschak 也告诉我，他试了无数方法让模型擅长写间隔重复卡片——上下文学习、监督微调、检索增强——就是达不到他的满意标准。所以即便在纯语言领域，要从这些模型中获得大量经济价值也很难。

Andrej Karpathy

我同意。我不是说所有文本任务都简单。代码确实很有结构，而文本更自由、更有不确定性。而且代码很难，所以人们即便从简单的 LLM 知识中也感到很强的赋能。但这不意味着所有文本任务都简单。

超级智能与失控

Dwarkesh Patel

你怎么看超级智能？你预期它的感觉会和普通人类或人类组织有质的不同吗？

Andrej Karpathy

我把它看作社会自动化的延续，是计算趋势的外推。会有越来越多自主实体在做数字工作，最终也做物理工作。大致就是自动化。它确实会看起来非常陌生，因为你可以在计算机集群上运行这些东西，速度快得多。

让我开始紧张的一些场景是这种渐进式的失控和理解丧失——我觉得这实际上是最可能的结果。我们会逐渐把这些东西到处部署，理解它们的人越来越少，然后逐渐失去对正在发生之事的控制和理解。

Dwarkesh Patel

失去控制和失去理解不一定是一回事吧？公司董事会可能理解不多但权力很大。

Andrej Karpathy

你说得对。但我预期两者都会失去。如果我要写科幻小说，不是一个单一实体接管一切那种——而是多个相互竞争的实体变得越来越自主，有些跑偏了，其他的去对抗它们。一种完全自主活动的大杂烩，我们已经委托出去了。不是因为它们比我们聪明所以失去控制——是因为它们在相互竞争，竞争中涌现出来的东西导致了失控。

智能爆炸与经济增长

Dwarkesh Patel

如果你有 AGI，它应该能做你做的事。拥有一百万个你在并行工作能大大加速 AI 进步吗？你预期会看到智能爆炸吗？

Andrej Karpathy

我预期会——但这是「一切照旧」，因为我们已经处于智能爆炸中了，而且已经持续了几十年。如果你看 GDP，基本上就是一条指数曲线——是所有行业方面的指数加权和。一切都在被渐进自动化，已经持续了几百年。工业革命是物理组件的自动化，编译器是早期的软件自动化。所以我觉得我们一直在递归式自我改进和爆炸。

LLM 让工程师更高效地构建下一轮 LLM——所有工程师有 Google 搜索、IDE、自动补全、Claude Code 等等，这些都是同一个加速过程的一部分。

我有一段时间试图在 GDP 中找到 AI 的信号，然后我去看了其他我认为很具变革性的技术——比如计算机、手机——你在 GDP 里找不到它们。GDP 是同一条指数曲线。早期 iPhone 没有 App Store，没有现在的各种功能——所以即使我们觉得 2008 年是重大变革，其实不是。一切都太分散了，太缓慢地扩散了，最终都平均到同一条指数曲线里。AI 也会一样。

Dwarkesh Patel

你是说增长率不会变化——智能爆炸只是帮助我们维持在 2% 增长轨道上，就像互联网帮助我们维持 2% 增长一样？

Andrej Karpathy

我的预期是保持同样的模式。

Dwarkesh Patel

我的预期是它会爆发。因为我觉得真正的 AGI——不是 LLM 编码机器人，而是真正能替代服务器中一个人类的东西——和其他生产力提升技术质量上不同，因为它是劳动力本身。我们活在一个极度劳动力受限的世界。如果你有几十亿额外的聪明人在发明东西、融入经济、创办公司——这感觉和单一技术质量上不同。就像如果地球突然多了一百亿人。

Andrej Karpathy

但计算就是劳动力——计算机让很多工作消失了，因为计算机在自动化一堆数字信息处理。自动驾驶也是计算机在做劳动。这些已经在发生了。

Dwarkesh Patel

但如果你看工业革命之前到现在的趋势，有一个超指数增长——从 0% 增长到几千年前的 0.02%，到现在的 2%。你是说 AI 不会再带来一次类似的增长率跳跃吗？

Andrej Karpathy

我理解你的观点——某种认知能力上存在过剩待释放的需求，一旦新技术跨过门槛就会填补这个过剩。但我仍然觉得人们在预设某种离散跳跃——某种解锁在等着我们，突然我们就在数据中心里有了天才。我觉得这种离散跳跃在历史上基本没有先例。

Dwarkesh Patel

工业革命就是这样一个跳跃——从 0.2% 增长到 2% 增长。工业革命的关键恰恰是它不是魔法——如果你放大来看 1770 年或 1870 年，并没有某个关键发明。但经济确实转移到了一个进步快得多的状态。

Andrej Karpathy

嗯，也许你说的对——也许会有一个类似工业革命的过剩释放。我对这个观点不强烈，愿意被说服。

自动驾驶的启示

Dwarkesh Patel

你在 Tesla 从 2017 年到 2022 年领导自动驾驶。你亲眼看到了从酷炫演示到现在数千辆车真正自主行驶的进步。为什么这需要十年？

Andrej Karpathy

首先，这远没有完成。自动驾驶最早的演示可以追溯到 1980 年代——CMU 在 1986 年就有一辆自动驾驶卡车。快进到我加入 Tesla 的时候，大约 2014 年我坐过一次 Waymo 的早期演示车——在 Palo Alto 周围完美地开了一圈。我当时觉得快要实现了。但还是花了很长时间。

对于某些类型的任务和工作——失败成本太高的那种——存在一个很大的「演示到产品」的鸿沟。演示很容易，产品很难。我觉得在软件工程中这个性质也存在——写真正的生产级代码时，任何错误都可能导致安全漏洞，数亿人的个人信息被泄露。所以我觉得在软件中人们也应该小心——有些后果几乎是无上限的糟糕。

基本上花时间的原因是这是一场「九的进军」——每个九需要相同的工作量。当你有一个演示，东西 90% 的时间能用——那只是第一个九。然后你需要第二个九、第三个九、第四个九、第五个九。我在 Tesla 待了五年左右，我们大概走过了两三个九，还有更多的九要走。

所以这些东西就是需要这么久。我对演示非常不感兴趣——无论谁给我看什么演示，我都极度不为所动。即使你能和它交互也还不够——你需要真正的产品。它面对现实时会遇到各种挑战和需要修补的行为角落。这是九的进军，每个九需要恒定的工作量。

而且自动驾驶汽车远没有完成。Waymo 的部署仍然很有限——因为它们不经济。它们建造了某种活在未来的东西，但不得不把未来拉回来使它不经济。而且当你看这些没人驾驶的车时，有些误导性——实际上有非常复杂的运营中心，有人在那里参与这些车的运作。在某种意义上我们没有去掉那个人，只是把他移到了我们看不到的地方。

我觉得 Tesla 采取了在我看来更具扩展性的策略，团队做得非常好。我不想把自动驾驶说成「花了十年」的事情——因为起点是 1980 年而不是 10 年前，而且终点还没到。

Dwarkesh Patel

自动驾驶花这么久的一大原因是需要解决基本感知和表示能力、让模型有一些常识以便能泛化。但 LLM/VLM 免费获得了这些——我们不用解决这些基本表示问题。所以在不同领域部署 AI 有点像把自动驾驶扩展到一个新城市——困难但不是十年级别的任务。

Andrej Karpathy

我不确定我完全同意。我不知道我们免费获得了多少——LLM 仍然相当容易出错，仍然有很多理解上的空白需要填补。我不认为我们完全开箱即用地得到了神奇的泛化能力。

另外我也想强调：自动驾驶仍然远未完成。Waymo 的车不能去城市的所有地方。我个人很喜欢 Waymo，一直在坐，但人们有时对进展有些天真。

Dwarkesh Patel

那当前这波 AI 基建投资——要在一两年内把世界上的算力翻 10 倍，到十年末可能翻 100 倍以上——如果 AI 的应用不如有些人天真预测的那么多，那是不是算力建设过度了？

Andrej Karpathy

我知道我听起来很悲观，但我其实是乐观的——我觉得这些东西是可行的、可解决的。我之所以听起来悲观，只是因为我在推特时间线上看到太多对我来说毫无意义的东西。我觉得很多就是在为融资造势、把注意力转化成金钱。

但我对技术整体非常看好。进步一直很快。我不认为有过度建设——Claude Code 或 OpenAI Codex 这样的东西一年前都不存在，这是一年前不存在的奇迹级技术。需求会很大。我只是在回应那些不断错误地说出非常快时间线的人——15 年来很多非常有声望的人一直在犯这个错误。我希望我们对现实保持校准，不要在此基础上做出地缘政治层面的误判。

智能的进化

Dwarkesh Patel

你推荐给我 Nick Lane 的书，我也觉得超级有趣所以采访了他。经过 20 年的 AI 研究，你现在对智能有了更具体的感觉——你更惊讶还是不那么惊讶，进化居然自发地偶然发现了智能？

Andrej Karpathy

我很喜欢 Nick 的书。关于智能及其进化——它确实出现得非常非常晚。我对它进化出来感到惊讶。想想那外面的一千颗类地行星会是什么样子——Nick 说他预期会有非常相似的生命形式，大部分是细菌。我直觉上觉得智能的进化应该是一个相当罕见的事件。

细菌存在了 20 亿年什么也没发生——说明从细菌跨越到下一步非常困难。多细胞动物大概存在了几亿年——也许是地球寿命的 10%。在那个时间尺度上也许不算太难。但我仍然觉得从动物发展出智能是令人惊讶的——我更预期的是动物只是越来越强壮之类的。走向智能确实是一个迷人的突破点。

Dwarkesh Patel

如果你认同 Sutton 的观点——到了松鼠智能就已经完成了 AGI 的大部分路程——那松鼠智能在寒武纪大爆发之后就出现了。这暗示一旦环境条件到位（大氧化事件），智能算法很快就出现了。也许它比看起来简单。

Andrej Karpathy

很难判断。也许可以看智能独立出现了多少次——灵长类智能和鸟类智能（乌鸦等极其聪明），它们的大脑结构相当不同。如果智能独立出现过几次，你也许会预期它更频繁。

Dwarkesh Patel

有个有趣的观点是：可扩展的算法在人类和鸟类中都出现了，但人类处在一个奖励边际智能增长的进化生态位——我们有手可以学工具使用，可以外包消化功能把更多能量给大脑，这启动了飞轮。但鸟的大脑更大的话就会从空中掉下来。海豚也是——在水里你没法有火，化学上水下能做的事比陆地上少得多。

Andrej Karpathy

我同意这个生态位的观点。你需要不可预测的环境——进化不能把算法直接写进你的权重。很多动物基本上是预设好的。人类需要在「测试时」自己想办法——在出生之后。你需要那种快速变化的环境，让进化无法预见什么会有效，所以必须创造智能来在测试时解决问题。

Dwarkesh Patel

Quentin Pope 有个有趣的博文：人类的「急剧起飞」——6 万年前我们似乎就有了今天的认知架构，但直到 1 万年前才有农业革命，然后是现代文明。那 5 万年在做什么？你在建文化脚手架——代际间积累知识的能力。这个能力在 AI 训练中是免费获得的——重新训练模型时它可以在前一代的基础上继续。

Andrej Karpathy

也是也不是。LLM 并没有真正的文化等价物——也许我们给了它们太多东西反而不激励它们去创造文化。LLM 不写书给其他 LLM 读、不被它启发或震惊。这些都没有等价物。

我觉得多 Agent 领域有两个强大的想法还没被真正实现。第一个是文化——LLM 自己的、不断增长的知识库。第二个更像自博弈（Self-Play）的强大想法——进化本身就是大量竞争在驱动智能和进化。AlphaGo 通过自博弈变得非常强。但没有 LLM 自博弈的等价物——为什么一个 LLM 不能创造一堆题目让另一个 LLM 去解、然后不断出更难的题呢？

我觉得根本原因是这些模型目前还像幼儿园学生——我知道它们能做 PhD 考试，但在认知上仍然感觉像幼儿园或小学生。它们是学者症候群式的神童——有完美的记忆力，能令人信服地产出各种看起来很好的东西。但我仍然觉得它们并不真正知道自己在做什么，在我们需要打勾的那些认知能力上还有很多空白。

教育与 Eureka

Dwarkesh Patel

你可以去创办另一家 AI 实验室来解决这些问题。你现在在做什么？为什么不做 AI 研究本身？

Andrej Karpathy

我对 AI 实验室在做的事情觉得有一定的确定性。我能帮上忙，但不确定我能带来独特的改进。我个人最大的担忧是：这些东西在人类这边发生了什么——人类会因此被边缘化。我不只关心 AI 会建造的戴森球——我关心人类会怎样。我想要人类在这个未来过得好。

我最害怕的情景可能是电影《机器人总动员》或《蠢蛋进化论》那种——人类被搁置在一边。我想要人类变得更好，而不是更差。我觉得通过教育可以实现这一点。

所以 Eureka 试图建造的——最简单的描述是「星际舰队学院」。一个面向前沿技术的精英机构。

Dwarkesh Patel

我很想了解你怎么思考技术和科学内容的教学——因为你是这方面的世界级大师。

Andrej Karpathy

教育会因为 AI 而发生根本性变化，它需要被重新设计。我觉得我们还很早期——很多人会做显而易见的事情，比如让 LLM 回答问题。我觉得这有帮助但仍然感觉粗糙。

我想要的是真正的家教体验。我最近在学韩语——我经历了自学、小班课、一对一家教。我的家教非常好。她从一小段对话就能立刻理解我作为学生处在什么阶段、知道什么不知道什么，能精确地探测我的世界模型。目前没有任何 LLM 能做到这一点——差得远。但一个好的家教能做到。

一旦她理解了我的水平，她就完美地给我提供了恰好在我能力边缘的内容——始终适度挑战，不太难也不太简单。我感觉我自己是唯一的学习瓶颈。不是我找不到知识，不是讲解不好——只是我自己的记忆和理解能力。

这就是我想为所有人实现的。当前的能力还不足以构建这种 AI 家教——这个标准太高了。但时机会到来的。

现在我在建的更传统一些——我在建第一个课程，目标是做出一个最先进的、显然是最好的学 AI 的地方。NanoChat 就是 LLM 101n 课程的毕业项目。我还需要建大量中间内容，招一个小团队的助教来完整地构建这个课程。

我有一个很技术性的愿景：教育是一个构建「知识坡道」的技术问题。NanoChat 就是一条知识坡道——一个超级简化的全栈实现。如果你给某人这个产物让他们看完，他们会学到大量东西。我想要大量的「每秒 Eureka 时刻」——每秒的理解量。这是一个技术问题：怎么高效构建这些坡道，让人永远不会卡住，一切都恰到好处地不太难也不太简单。

Dwarkesh Patel

短期内这是不是主要靠你把 AI 讲清楚的能力——编码在课程材料里——而不太依赖 AI 自动化？

Andrej Karpathy

你需要校准于当前存在的能力。如果你现在去 ChatGPT 说「教我 AI」——它会给你一些粗糙的东西。AI 现在写不出 NanoChat。但我正在和 AI 协作创建所有材料——和我之前在 Stanford 建 CS231N 时相比，现在快多了，AI 帮我做大量无聊的工作。但创造性地组织内容仍然需要我在场。

我想象一个发展路径：现在是招教授来和 AI 以及团队合作，构建最先进的课程。然后逐渐有些助教可以变成 AI——把所有课程材料给 AI，它就能为学生服务一个不错的自动化助教来回答基本问题。但课程的整体架构仍然需要教授。也许未来某天我自己也不再需要了、AI 能比我做得更好，但那需要时间。

Dwarkesh Patel

你觉得教育和保持人类对 AI 的控制力有什么关系？

Andrej Karpathy

长期来看可能确实是一场「注定败北」的比赛。但我觉得人们能走得比现在远得多——我们仅仅触及了人类大脑能力的皮毛。人们会从材料面前弹开，因为太难或太简单。如果有完美的 AI 家教，也许你能走得极远——每个人都能说五种语言、掌握所有本科基础课程。

这和健身房文化有完美的类比。100 年前没人会像现在这样「练出线条」。系统化训练、健身房——这些让普通人拥有了自然状态下不会有的身体能力。我在想象认知领域的类似事情——在更多领域、更深入、更快速地学习。

AGI 之前，教育是有用的——因为人们需要赚钱，而这就是现在在行业里赚钱的方式。AGI 之后，教育是有趣的——就像人们去健身房一样。我们不需要他们的体力了，但他们仍然去，因为有趣、健康、而且——你练出六块腹肌看起来不错。我觉得教育会以同样的方式发展——你去上学就像去健身房一样。

教学方法论

Dwarkesh Patel

如果你要给另一个领域的教育者提建议，让他们做出你那种水平的 YouTube 教程，你会说什么？

Andrej Karpathy

这是个很广泛的话题，我可能有 10 到 20 个半有意识在用的技巧。高层来说，这来自我的物理学背景。我真的非常喜欢物理学训练。我一直觉得每个人都应该在早期教育中学物理——不是为了积累知识，而是为了启动大脑。物理独特地训练你的大脑做一些极其有价值的事情：构建模型和抽象，理解存在一阶近似描述了系统的大部分，然后还有二阶、三阶项可能存在也可能不存在。

「假设一头球形的牛……」——每个人都笑这个，但这实际上是绝妙的思维方式。它非常可泛化——动物在很多方面确实可以被近似为球体。有一本很好的书叫 Scale，是一个物理学家写的关于生物学的书——你能得到动物的非常有趣的近似和标度律，比如心跳和动物体型的关系。你可以把动物当作一个体积来讨论散热——散热按表面积增长（平方），产热按体积增长（立方）。

因为这种训练，我总是试图找到一切事物的一阶项和二阶项。我脑中有一团纠缠的想法和知识，我在试图找到真正重要的东西——一阶成分是什么？怎么简化？怎么有一个简单的东西展示那个核心，然后再叠加其他项？

Micrograd 就是一个好例子——100 行 Python 代码展示反向传播。你可以用加法和乘法等乐高积木构建计算图，做前向传播和反向传播。这 100 行代码就是理解神经网络训练所需的一切。其他一切都只是效率——你需要张量、内存布局、内核编排等等。但核心的智识内容就是 Micrograd——链式法则的递归应用来求导，以优化任何可微函数。

我喜欢发现这些「最小项」并把它们呈上盘子。我觉得教育就是最有趣的智力问题——你有一团纠缠的理解，试图把它铺排成一条坡道，每样东西只依赖它前面的东西。

Dwarkesh Patel

你的 Transformer 教程从 bigram 开始——字面意思就是一个查找表，「上一个词是这个，下一个词是那个」。然后从查找表走到 Transformer，每一步都有动机——为什么要加这个？它解决什么问题？

Andrej Karpathy

没错，你在呈现解决方案之前先呈现痛点。我不会在给你机会自己猜之前就告诉你答案——因为如果你先自己试了，当我给出答案时你有更深的体会。它最大化了每个新事实带来的知识量。

Dwarkesh Patel

为什么真正的专家往往不擅长向新手解释自己的领域？

Andrej Karpathy

知识的诅咒——这是真实存在的现象，我自己也深受其害。你把某些东西当成理所当然，无法设身处地地站在初学者的立场。

一个非常有效的技巧：如果有人写了论文或博客文章或公告——100% 的情况下，他们在午餐时给你口头解释时的叙述，比书面版本更好理解，而且实际上也更准确。人们有一种用最抽象、最充满术语的方式来解释事物的偏好，在正文前清嗓子四段。但一对一交流时有某种东西迫使你直接说重点。

我在读博时就经常遇到这种情况——你读了某人的论文，苦苦理解。然后在会议后的啤酒桌上问他们论文讲了什么，他们用三句话就完美捕捉了论文的精髓。你根本不需要读论文。但只有在啤酒桌上那种随意的场合他们才会这么说——为什么那不就是论文本身呢？

Dwarkesh Patel

从学生角度——如果你没有一个 Karpathy 来给你做讲解，你自己学习不熟悉领域的材料有什么策略？

Andrej Karpathy

这基本上是个痛苦的过程。但有一些帮助我的东西。

按需学习——深度优先——非常好。你在试图完成一个能得到回报的项目，按需学习。和广度优先交替——「相信我你以后会用到这个」那种。我喜欢那种你能从中实际获得回报的学习。

另一个极其有用的方式是给别人解释——因为我发现如果我不真正理解某个东西，我就解释不了。这迫使你面对自己理解中的空白、与它们和解。我喜欢重新解释东西——人们应该更多地这么做。这迫使你操作知识、确保你真正懂你在说什么。
